## 정의
고객의 피드백을 얻기 위한 최소 기능을 갖춘 제품, MVP
- 가설을 검증하고 시장 반응을 살피기 위해 핵심 기능만을 포함하여 빠르게 출시하는 제품
- 초기 단계에서의 시행착오를 줄이고 비즈니스 방향성(Pivot)을 결정하는 기준 제공

## 구성요소
- 최소 기능(Minimum): 완성도가 아닌 가치 검증에 필요한 필수 기능만 포함
- 실행 가능성(Viable): 실제 사용자가 가치를 느끼고 문제를 해결할 수 있어야 함
- 가설 검증: 고객의 니즈가 실제로 존재하는지 데이터를 통해 확인
- Build-Measure-Learn: 만들고 측정하고 배우는 린 스타트업의 핵심 주기 적용
- Pivot/Persevere: 검증 결과에 따라 방향을 틀거나(Pivot) 유지(Persevere) 결정

## 구성도 요소
* 아키텍처: 피라미드 구조 (Functional -> Reliable -> Usable -> Emotional) 중 세로로 얇게 잘라낸 형태
* 관계: 완벽한 제품이 아닌 '배움'을 위한 도구로서의 역할 강조
```
[Idea] -> [Build] -> [MVP] -> [Measure] -> [Data] -> [Learn]
  ^                                                    |
  +------------------- [Improve / Pivot] <-------------+
```

## 키워드
* 린 스타트업, 가설 검증, 피벗(Pivot), 얼리 어답터, 빠른 출시

## 암기법
