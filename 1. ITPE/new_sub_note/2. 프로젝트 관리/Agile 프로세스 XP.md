## 정의
개발자 중심의 기술적 실천을 강조하는 애자일 방법론, XP
- '익스트림(Extreme)'이라는 이름처럼 프로그래밍의 좋은 관행들을 극단적으로 실천하는 방식
- 고객의 요구에 유연하게 대응하면서 고품질의 소프트웨어를 빠르게 개발하는 것이 목표

## 구성요소
- 5대 가치: 의사소통, 단순성, 피드백, 용기, 존중
- 실천 방식(Practices):
    - TDD(Test Driven Development): 테스트 코드 먼저 작성
    - Pair Programming: 두 명이 한 조로 개발
    - Refactoring: 기능 유지하며 코드 구조 개선
    - CI(Continuous Integration): 수시로 코드 통합 및 빌드
    - Collective Ownership: 팀원 전체가 모든 코드에 책임 공유
- 주기: 작은 릴리스(Small Releases)를 통해 잦은 배포 수행

## 구성도 요소
* 프로세스: User Story → Planning → Iteration (TDD/Pair) → Release → Feedback
* 관계: 개발 생산성과 품질을 동시에 높이기 위한 엔지니어링 중심 접근
```
[User Story] -> [Spike] -> [Release Plan] -> [Iteration]
                                                |
                                      [TDD / Pair / CI]
```

## 키워드
* TDD, 페어 프로그래밍, 리팩토링, 지속적 통합, 5대 가치

## 암기법
* 단순용피의 (단순성, 순발력-용기, 피드백, 의사소통 - 가치)
* 테짝리공지 (TDD, 짝프로그래밍, 리팩토링, 공동소유, 지속적통합 - 실천)
