## 정의
흐름 시각화와 워크플로우 최적화 중심의 애자일 기법, 칸반
- 작업의 흐름을 가시화하고 진행 중인 작업(WIP)을 제한하여 생산성을 극대화하는 방식
- 특정 시간 단위(Sprint)보다 작업의 연속적인 흐름과 속도(Velocity) 관리에 집중함

## 구성요소
- 칸반 보드(Kanban Board): To-Do, In-Progress, Done 등 작업 상태를 시각화
- WIP 제한(Work In Progress Limit): 동시에 진행할 수 있는 작업의 수를 강제로 제한 (병목 제거)
- 리드 타임(Lead Time): 요청부터 완료까지 걸리는 전체 시간
- 사이클 타임(Cycle Time): 실제 작업 시작부터 완료까지 걸리는 시간
- 누적 흐름도(CFD): 작업 상태별 수량 변화를 시각화하여 흐름의 안정성 확인

## 구성도 요소
* 원칙: 시각화(Visualize), WIP 제한(Limit WIP), 흐름 관리(Manage Flow), 명확한 정책(Explicit Policies)
* 관계: 스크럼과 결합하여 '스크럼반(Scrumban)'으로 활용되기도 함
```
[Backlog] | [In-Progress (WIP:3)] | [Testing] | [Done]
   [Task] |      [Task][Task]    |  [Task]   | [Task]
   [Task] |      [Task]          |           | [Task]
```

## 키워드
* 시각화, WIP 제한, 리드 타임, 사이클 타임, 풀 시스템(Pull System)
