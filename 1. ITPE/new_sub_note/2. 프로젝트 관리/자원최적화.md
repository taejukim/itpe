## 정의
자원의 가용성에 맞춰 일정을 조정하고 효율성을 극대화하는 기법, 자원최적화
- 한정된 자원의 제약 조건을 고려하여 프로젝트 일정을 현실화하거나 자원 활용의 편차를 줄이는 활동
- 일정 지연을 감수하더라도 자원 제약을 준수하거나, 일정 내에서 자원을 평활하게 배분함

## 목적
- 제한된 인적·물적 자원을 효율적으로 배분하고 활용하여 자원 과부하를 방지하고 일정 지연 리스크 해소
- 자원 평준화 및 평탄화 기법을 통해 자원 사용의 안정성을 확보하고 프로젝트 비용 효율성 증대
- 활동별 자원 요구사항과 가용성을 정교하게 조율하여 프로젝트 수행의 생산성과 품질 극대화
- 
## 구성요소
- 자원 평준화(Resource Leveling): 자원 과부하를 해소하기 위해 일정 조정
- 자원 평탄화(Resource Smoothing): 프로젝트 종료일은 유지하면서 자원 사용량을 균등화
- 자원 할당: 프로젝트 활동에 필요한 자원 배정
- 자원 가용성 분석: 자원의 가용 시간과 용량 파악
- 자원 충돌 해결: 동시에 필요한 자원의 우선순위 결정
- 자원 모니터링: 실제 자원 사용량 추적 및 분석

## 구성도 요소
* 메커니즘: 과부하 발생(Overload) → 최적화 기법 적용 → 안정적 자원 활용
* 관계: 일정 개발 단계에서 [[CPM(임계경로 관리)]] 분석 후 자원 제약을 반영하기 위해 적용
```
[Peak Load] ---- (Leveling/Smoothing) ----> [Balanced Load]
      |                                           |
(Overallocation)                          (Stable Allocation)
```

## 키워드
* 레벨링, 스무딩, 자원 제약, 주공정 변화, 여유 시간 활용

## 암기법
* 평준평활 (자원 평준화, 자원 평탄화)
* 지연유지 (평준화는 지연 가능, 평탄화는 일정 유지)
* 평평할가충모: 평준화, 평탄화, 할당, 가용성, 충돌 해결, 모니터링
