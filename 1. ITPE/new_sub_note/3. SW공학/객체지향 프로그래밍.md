## 정의
데이터와 기능을 하나의 단위인 객체로 묶어 관리하는 프로그래밍 패러다임, OOP
- 현실 세계의 개체(Entity)를 속성(Data)과 행위(Method)를 가진 객체로 모델링
- 코드의 재사용성, 유지보수성, 확장성을 높이기 위해 [[캡슐화(encapsulation)]], 상속, [[다형성(Polymorphism)]] 등을 활용함

## 구성요소
- 클래스(Class): 객체를 생성하기 위한 틀(붕어빵 틀), 청사진
- 객체(Object): 클래스로부터 생성된 실체(붕어빵), 인스턴스
- 속성(Attribute): 객체가 가진 데이터 (변수)
- 메서드(Method): 객체가 수행하는 기능 (함수)
- 메시지(Message): 객체 간의 상호작용을 위한 통신 수단

## 구성도 요소
* 핵심 원칙: [[캡슐화(encapsulation)]], 상속, [[다형성(Polymorphism)]], [[추상화(abstraction)]] (객체지향 4대 특징)
* 관계: 객체 간의 관계(연관, 의존, 일반화, 실체화)를 통해 복잡한 시스템 구성
```
[Class Name]
------------
- Attribute
------------
+ Method()
```
