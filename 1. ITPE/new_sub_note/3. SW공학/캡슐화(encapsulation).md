## 정의
데이터와 기능을 하나로 묶고 내부 구현을 외부에 숨기는 기법, 캡슐화
- 객체의 속성과 메서드를 하나의 틀 안에 결합하고, 외부의 직접적인 접근을 제한하는 방식
- 결합도를 낮추고 응집도를 높여 객체의 독립성과 유지보수성을 향상시킴

## 구성요소
- 멤버 변수: 객체의 상태를 저장하는 데이터 (Private 권장)
- 멤버 메서드: 객체의 동작을 수행하는 함수
- 접근 제어자(Access Modifier): Public, Private, Protected 등을 통한 접근 범위 설정
- 인터페이스: 외부에서 객체의 기능을 이용할 수 있도록 노출된 통로 (Public)
- 정보 은닉: 캡슐화의 결과로 내부 상세 구현이 숨겨지는 상태

## 구성도 요소
* 아키텍처: 데이터는 안쪽에, 메서드는 바깥쪽에 배치하여 데이터를 보호하는 구조
* 관계: 객체의 내부 구현이 변경되어도 외부 인터페이스가 유지되면 사용하는 쪽에서는 수정이 필요 없음
```
+---------------------+
|      (Methods)      |
|  +---------------+  |
|  |  (Data/Vars)  |  |
|  +---------------+  |
+---------------------+
```

## 키워드
* 정보 은닉, 접근 제어자, 낮은 결합도, 높은 응집도, 인터페이스

## 암기법
* 캡정 (캡슐화는 [[정보은닉(Information Hiding)]]을 동반함)
* 데기하 (데이터와 기능을 하나로)
