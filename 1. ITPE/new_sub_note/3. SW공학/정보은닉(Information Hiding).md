## 정의
객체의 내부 구현 상세를 외부에 노출하지 않는 설계 원칙, 정보은닉
- [[캡슐화(encapsulation)]]의 핵심 목표로, 객체의 내부 데이터를 직접 접근하지 못하게 차단하고 정해진 경로로만 소통
- 내부 로직이 변경되어도 이를 사용하는 외부 코드에 영향을 주지 않아 파급 효과(Side Effect)를 방지함

## 구성요소
- 접근 제어자: Private(내부만), Public(외부허용), Protected(상속관계), Default(패키지내)
- Getter/Setter: 내부 변수에 접근하기 위한 공개된 메서드 (데이터 무결성 유지)
- 인터페이스 분리: 필요한 기능만 외부에 노출하고 나머지는 숨김
- 모듈화: 내부 동작은 모듈 내부에 감추고 인터페이스만 제공
- 결합도 감소: 객체 간의 의존성을 인터페이스 수준으로 제한

## 구성도 요소
* 메커니즘: 외부 객체 -- (Public Method) --> [객체 경계] -- (Private Data)
* 관계: [[캡슐화(encapsulation)]]는 수단이고, 정보은닉은 그 결과로 얻어지는 상태임
```
[External] -> [Interface(Public)] -> [Implementation(Private)]
      |               |                       |
      +----- (Access Blocked) ----------------+
```

## 키워드
* 접근 제어자, 캡슐화, 사이드 이펙트 방지, 낮은 결합도, 데이터 보호

## 암기법
* 은접 (정보은닉은 접근제어자로 실현)
* 프프프 (Private, Protected, Public)
