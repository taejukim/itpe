## 정의
상위 클래스의 속성과 기능을 하위 클래스가 물려받아 재사용하는 기법, 상속성
- 부모 클래스(Super Class)의 특징을 자식 클래스(Sub Class)가 공유하도록 설계하는 방식
- 코드의 중복을 제거하고 계층적인 구조를 통해 시스템을 체계적으로 확장함

## 구성요소
- 부모 클래스: 공통적인 속성과 메서드를 정의한 상위 모델
- 자식 클래스: 부모로부터 상속받아 새로운 기능을 추가하거나 기존 기능을 수정(오버라이딩)한 모델
- Is-a 관계: "자식은 부모의 일종이다"라는 논리적 관계 성립 (예: 자동차는 탈것이다)
- 재사용: 부모의 코드를 다시 작성하지 않고 그대로 사용
- 확장: 부모의 기능을 기반으로 특화된 기능 추가 (Extends)

## 구성도 요소
* 아키텍처: 일반화(Generalization)와 구체화(Specialization)의 계층 구조
* 관계: 상속이 너무 깊어지면 결합도가 높아져 유지보수가 어려워질 수 있음 (상속보다 합성을 권장하기도 함)
```
     [Parent Class]
           ^
           | (extends)
     [Child Class]
```

## 키워드
* 재사용성, 일반화/구체화, Is-a 관계, 오버라이딩, 확장성

## 암기법
* 부자상속 (부모와 자식 관계)
* 일구 (일반화, 구체화)
