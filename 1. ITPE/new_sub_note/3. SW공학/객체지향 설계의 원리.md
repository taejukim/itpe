## 정의
유연하고 유지보수가 용이한 객체지향 설계를 위한 5가지 원칙, SOLID
- 로버트 마틴(Robert C. Martin)이 제안한 5대 원칙으로, 객체지향 기술의 장점을 극대화함
- 코드의 변경이 용이하고 결합도를 낮추어 소프트웨어 품질을 높이는 설계 가이드라인

## 구성요소
- SRP(Single Responsibility): 단일 책임 원칙. 클래스는 하나의 책임만 가져야 함
- OCP(Open-Closed): 개방-폐쇄 원칙. 확정에는 열려 있고 변경에는 닫혀 있어야 함
- LSP(Liskov Substitution): 리스코프 치환 원칙. 자식은 부모를 언제나 대체할 수 있어야 함
- [[ISP]](Interface Segregation): 인터페이스 분리 원칙. 사용하지 않는 인터페이스에 의존하지 않아야 함
- DIP(Dependency Inversion): 의존 역전 원칙. 구체화가 아닌 [[추상화(abstraction)]]에 의존해야 함

## 구성도 요소
* 아키텍처: SOLID 원칙을 준수하여 설계된 클래스 다이어그램
* 관계: 디자인 패턴(Design Patterns)의 이론적 기초가 됨
```
[Client] -> [Interface (Abstraction)]
                   ^
                   |
           [Implementation (Concrete)]
```

## 키워드
* SRP, OCP, LSP, [[ISP]], DIP, 낮은 결합도, 높은 응집도
