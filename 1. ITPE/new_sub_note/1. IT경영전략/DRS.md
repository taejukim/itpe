## 정의
재해 복구 계획을 실현하기 위한 기술적 시스템, DRS
- 정보시스템의 안정성을 보장하기 위해 ==주 센터==와 떨어진 지역에 구축한 복구 시스템
- 구축 수준에 따라 [[RTO, RPO]]가 결정되며 무중단 서비스를 지향함
특징) 실시간 데이터 복제, ==Fail-over/Fail-back==, 고가용성 보장

## 구성요소
- Mirror Site: 실시간 동기 복제, RTO=0 (Active-Active)
- Hot Site: 실시간 비동기 복제, RTO=수시간 이내 (Active-Standby)
- Warm Site: 주기적 백업 데이터 전송, RTO=수일 이내
- Cold Site: 장소만 확보 및 장비 미설치, RTO=수주 이내
- 백업 센터: 데이터 보관 및 시스템 복원을 위한 물리적 거점

## 주요 기술요소
- 데이터 복제: 동기(Sync)/비동기(Async) 방식, 스토리지 기반 복제, CDP
- 네트워크: DWDM, 전용선, VPN, 글로벌 서버 부하 분산(GSLB)
- 센터 전환: Fail-over(재해 발생 시 전환), Fail-back(정상 복구 시 회귀)
- 가용성: HA(High Availability), FT(Fault Tolerant), IP-SAN

## 구성도 요소
* 아키텍처: 데이터 복제 기술(Storage mirroring, Database replication) 중심
* 관계: 거리와 대역폭에 따라 동기(Synchronous)/비동기(Asynchronous) 방식 선택
```
[ 주 센터 (Primary) ]                  [ DR 센터 (Secondary) ]
    ┌───────────────────────┐              ┌───────────────────────┐
    │  운영 서버 및 데이터베이스  │              │  대기 서버 및 복제 데이터  │
    └──────────┬────────────┘              └───────────┬───────────┘
               │                                       │
               │        <─── 데이터 복제 ───>          │
               │      (Mirroring/Replication)          │
               │                                       │
    [ 재해 발생 시 (Disaster) ] ──────────▶ [ 서비스 전환 (Failover) ]
               │                                       │
      RPO (복구 지점) 기준                    RTO (복구 시간) 기준
    "얼마나 이전 데이터인가?"                "얼마나 빨리 복구하는가?"
```

## 운영 형태
- 독립 구축형: 자체 센터 보유 및 운영
- 상호 계약형: 타 기관과 상호 센터 활용 협약
- 공동 구축형: 여러 기관이 공동으로 센터 구축 및 분담
- 외주 위탁형: 전문 DR 서비스 업체 활용 ([[DRaaS]])
