## 정의
문제 해결을 위한 중복 없는 최적의 분석 기법, MECE/LISS
- MECE: 어떤 사항을 중복 없이(Exclusive), 누락 없이(Exhaustive) 부분으로 나누는 원리
- LISS: 중요한 포인트를 누락 없이 파악하되 전체의 핵심을 논리적으로 구성하는 기법

## 구성요소
- Mutually Exclusive: 개별 항목들 간에 겹치는 부분이 없음 (중복 제거)
- Collectively Exhaustive: 전체를 합쳤을 때 빠진 부분이 없음 (누락 방지)
- 논리적 분해: 피라미드 구조(Pyramid Structure)를 통한 단계적 상세화
- 핵심 추출: 단순한 나열이 아닌 해결책 중심의 유의미한 그룹화

## 구성도 요소
* 아키텍처: 전체 문제 → 1단계 분해 → 2단계 상세화 (Tree 구조)
* 관계: 문제 정의 단계에서 로직 트리(Logic Tree) 작성 시 기본 원칙으로 적용
```
      [전체 문제]
      /    |    \
   [A]    [B]    [C]  (Exclusive: A!=B!=C)
  / \    / \    / \
[..][..][..][..][..][..] (Exhaustive: A+B+C=Total)
```

## 키워드
* 로직 트리, 피라미드 구조, 중복 제거, 누락 방지, 문제 해결

## 암기법
* 중없누없 (중복 없이 누락 없이)
* 노빠중 (No 빠짐, 중복 없이)
