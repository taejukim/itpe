## 정의
재해 발생 시 IT 서비스 복구를 위한 실행 계획, DRP
- 정보시스템의 중단을 초래하는 재해 발생 시 이를 신속히 복구하기 위한 상세 절차서
- BCP의 기술적 하부 계획으로, 하드웨어, 소프트웨어, 네트워크, 데이터 복구 중심

## 구성요소
- 복구 전략: 미러/핫/웜/콜드 사이트 등 구축 방식 결정
- 비상연락망: 재해 선포 및 복구 조직 가동을 위한 통신 체계
- 복구 절차서: 각 시스템별 복원 단계 및 순서를 기술한 매뉴얼
- 테스트 및 훈련: 시나리오 기반의 정기적 모의훈련 계획
- 자원 관리: 복구에 필요한 백업 매체, 부품, 인력 확보 방안

## 구성도 요소
* 수행 단계: 재해 선포 → 비상 대응 → 시스템 복구 → 업무 복원 → 정상화(Failback)
* 핵심 지표: RTO(시간), RPO(데이터) 준수 여부 관리
```
[재해 발생] -> [재해 선포] -> [DR 센터 가동] -> [데이터 복원] -> [서비스 재개]
      |                                                        |
      +---------------------- [RTO 목표 준수] ------------------+
```

## 키워드
* 복구 시나리오, 비상대응팀, 백업 지침, [[DRS]] 연계

## 암기법
* 선대복원 (재해선포, 대응, 복구, 복원)
