## 정의
프로젝트 일정 단축기법
프로젝트 일정을 단축하기 위한 다양한 기법들, 일정 단축기법
- 프로젝트 완료일을 앞당기기 위해 사용하는 일정 압축 기법
- 각 기법은 장단점이 있으므로 프로젝트 상황에 맞게 선택하여 적용

## 구성요소
| 기법 | 설명 | 장점 | 단점 |
|------|------|------|------|
| 크래싱(Crashing) | 추가 자원 투입 | 일정 단축 효과 즉시 | 비용 증가 |
| 패스트 트래킹(Fast Tracking) | 순차 활동을 병렬 수행 | 비용 증가 없음 | 리스크 증가 |
| 범위 축소 | 범위를 줄여 일정 단축 | 빠른 효과 | 기능 제한 |
| 자원 교체 | 숙련된 자원으로 교체 | 생산성 향상 | 비용 증가 가능 |

## 구성도
```
[크래싱 (Crashing)]
                 추가 자원 투입
                      |
                      v
+---------+     +---------+     +---------+
| 활동 A  | --> | 활동 B  | --> | 활동 C  |
|  5일    |     |  10일   |     |  5일    |
+---------+     +----+----+     +---------+
                     |
                     v
               +----------+
               |  7일로   |  (3일 단축, 비용 증가)
               |  단축    |
               +----------+

[패스트 트래킹 (Fast Tracking)]
<순차 실행>                    <병렬 실행>
+---------+                    +---------+
| 활동 A  |                    | 활동 A  |--+
+---------+                    +---------+  |
    |                               +-------+
    v                               |
+---------+                    +----v----+
| 활동 B  |         -->        | 활동 B  | (동시 수행)
+---------+                    +---------+
    |                               |
    v                               v
+---------+                    +---------+
| 활동 C  |                    | 활동 C  |
+---------+                    +---------+
총 15일                        총 10일 (5일 단축)
                               * 리스크: 재작업 가능성

[기법 선택 기준]
+------------------+-------------------+
| 상황             | 권장 기법         |
+------------------+-------------------+
| 예산 여유 있음   | 크래싱            |
| 예산 제한적      | 패스트 트래킹     |
| 리스크 허용 가능 | 패스트 트래킹     |
| 리스크 회피      | 크래싱            |
+------------------+-------------------+
```
