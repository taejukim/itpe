## 정의
MSA와 Monolithic Architecture
마이크로서비스 아키텍처와 모놀리식 아키텍처의 비교, MSA vs Monolithic
- Monolithic: 모든 기능을 하나의 애플리케이션으로 구성하는 전통적 아키텍처
- MSA: 애플리케이션을 독립적인 작은 서비스로 분해하는 현대적 아키텍처

## 목적
- 비즈니스 요구사항과 프로젝트 규모에 적합한 아키텍처 선택을 위한 명확한 비교 기준 및 통찰 제공
- MSA 도입 시 서비스 간의 독립적 배포 및 확장을 통해 시스템의 유연성과 가용성을 확보하는 방안 제시
- 기존 모놀리식 구조의 한계를 분석하고 현대적 애플리케이션 환경에 최적화된 서비스 구조 설계 지원

## 구성요소
| 구분 | Monolithic | MSA |
|------|------------|-----|
| 배포 단위 | 단일 배포 | 서비스별 독립 배포 |
| 결합도 | 강한 결합 | 약한 결합 (Loose Coupling) |
| 데이터베이스 | 공유 DB | 서비스별 DB |
| 확장성 | 전체 확장만 가능 | 서비스별 독립 확장 |
| 기술 스택 | 단일 기술 스택 | 다양한 기술 스택 가능 |
| 장애 영향 | 전체 영향 | 해당 서비스만 영향 |

## 구성도
```
[Monolithic Architecture]
+--------------------------------------------------+
|                  애플리케이션                      |
|  +--------+  +--------+  +--------+  +--------+  |
|  |  UI    |  |  주문  |  |  결제  |  |  배송  |  |
|  +--------+  +--------+  +--------+  +--------+  |
|                     |                             |
|              +------+------+                      |
|              |  공유 DB    |                      |
|              +-------------+                      |
+--------------------------------------------------+
           단일 배포, 단일 확장

[Microservices Architecture (MSA)]
+----------+   +----------+   +----------+   +----------+
|   UI     |   |   주문   |   |   결제   |   |   배송   |
| Service  |   | Service  |   | Service  |   | Service  |
+----+-----+   +----+-----+   +----+-----+   +----+-----+
     |              |              |              |
     |         +----+----+    +----+----+    +----+----+
     |         |주문 DB  |    |결제 DB  |    |배송 DB  |
     |         +---------+    +---------+    +---------+
     |              |              |              |
     +-------+------+------+------+------+-------+
             |     API Gateway     |
             +---------------------+

[MSA 구성요소]
+------------------+-------------------------------------+
| API Gateway      | 외부 요청 라우팅, 인증, 로드밸런싱   |
| Service Registry | 서비스 등록 및 발견 (Eureka, Consul) |
| Config Server    | 중앙 설정 관리                      |
| Circuit Breaker  | 장애 격리 (Hystrix, Resilience4j)   |
| Service Mesh     | 서비스간 통신 관리 (Istio, Linkerd) |
+------------------+-------------------------------------+
```

## 키워드
- 모놀리식, 마이크로서비스, 독립적 배포, 약한 결합, 확장성, API Gateway, [[MSA 서비스 서비스 메쉬(Service Mesh)]]

## 암기법
- 모마독약확: 모놀리식, 마이크로서비스, 독립적 배포, 약한 결합, 확장성
