## 정의
DevOps
개발과 운영을 통합하여 소프트웨어 개발 및 배포 효율성을 향상시키는 문화와 방법론, DevOps
- 개발팀과 운영팀 간의 협업을 강화하여 빠르고 안정적인 소프트웨어 전달을 실현
- 자동화, 지속적 통합/배포, 모니터링을 통해 개발 생명주기를 최적화

## 목적
- 개발(Dev)과 운영(Ops) 팀 간의 장벽을 제거하고 협업 중심의 문화를 구축하여 전체 IT 서비스의 안정성과 속도 향상
- CI/CD, 자동화, IaC 등을 활용하여 소프트웨어 개발 생명주기 전반의 효율성을 극대화하고 배포 빈도 증대
- 실시간 모니터링과 피드백을 통해 장애를 신속하게 복구하고 지속적인 개선 시스템 구축

## 구성요소
| 구성요소 | 설명 |
|----------|------|
| 문화(Culture) | 개발과 운영 간 협업, 소통, 책임 공유 |
| 자동화(Automation) | 빌드, 테스트, 배포 프로세스 자동화 |
| 측정(Measurement) | 성과 지표 측정, 피드백 수집 |
| 공유(Sharing) | 지식, 도구, 경험 공유 |
| [[CI, CD]] | 지속적 통합, 지속적 배포 파이프라인 |
| IaC | Infrastructure as Code, 인프라 코드화 |
| 모니터링 | 애플리케이션, 인프라 실시간 모니터링 |
