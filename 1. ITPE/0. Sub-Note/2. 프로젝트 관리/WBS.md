#프로젝트관리 #범위관리 #필수

## 정의
프로젝트 목표 달성을 위한 ==인도물== 중심의 계층적 작업 분할 구조, WBS
- 프로젝트의 전체 범위를 관리 가능한 작은 구성 요소로 분할한 트리 구조
- 하위로 갈수록 작업이 상세해지며, 최하위 단위를 작업 패키지(Work Package)라고 함
- 
* 3 ~ 5레벨
## 키워드
* 작업 패키지, 100% 원칙, 인도물 중심, WBS 사전, ==범위 기준선==
## 암기법
* 인단부 (인도물, 단계, 부서 중심 분할)
* 원사 (100% 원칙, WBS 사전)
## 구성요소
- 패키지 
	- ==작업 패키지(Work Package)==: WBS의 최하위 수준으로 비용과 일정을 추정할 수 있는 단위
	- ==계획 패키지(Plan Package)== : 상세 작업 내용을 기술하지 못한 단위
- 분류 체계
	- Code of Account Identifier(CAID) : 작업의 고유 ID
	- WBS 사전(WBS Dictionary): 각 구성 요소에 대한 상세 정보(작업 내용, 담당자, 자원 등) 기술서
- 계층 구조: 대분류 → 중분류 → 소분류 → 작업 패키지 순으로 분할
- 코드 체계: 각 항목을 식별하기 위한 고유 번호 부여
- 100% 원칙: 하위 항목의 합은 반드시 상위 항목의 전체 범위와 일치해야 함
## 구성도 요소
* 분할 방식: 인도물 중심, 단계 중심, 부서 중심
* 관계: WBS는 일정(Network Diagram), 원가(Budget), 조직(OBS) 관리의 기초 데이터 제공
```
      [Project Root]
      /      |      \
 [Phase 1] [Phase 2] [Phase 3]
   /  \       |
[WP 1][WP 2] [WP 3] ...
```
