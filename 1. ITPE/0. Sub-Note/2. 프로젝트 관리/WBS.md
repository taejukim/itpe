## 정의
WBS (Work Breakdown Structure)
프로젝트 범위를 계층적으로 분해한 작업 분해 구조, WBS
- 프로젝트의 전체 범위를 관리 가능한 작은 작업 단위로 분해한 계층 구조
- 프로젝트 계획, 일정 수립, 비용 산정, 자원 할당의 기반이 되는 핵심 도구
- [[프로젝트 범위관리]]의 핵심 산출물로, [[프로젝트 관리 계획서]]에 포함됨

## 목적
- 프로젝트 전체 범위를 관리 가능한 수준의 작업 패키지로 세분화하여 계획 수립 및 통제의 기초 데이터 제공
- 작업 단위별 책임과 역할을 명확히 정의하고 일정, 비용, 자원 할당의 정확성을 높여 프로젝트 가시성 확보
- 누락되거나 중복된 작업(MECE)을 사전에 방지하여 프로젝트 범위 이탈 및 리스크 방지

## 구성요소
| 구성요소 | 설명 |
|----------|------|
| 최상위 레벨 | 프로젝트 전체 (Level 0) |
| 중간 레벨 | 주요 단계/산출물 (Phase/Deliverable) |
| 최하위 레벨 | 작업 패키지 (Work Package) |
| WBS 딕셔너리 | 각 작업 패키지의 상세 설명, 책임자, 일정, 비용 |
| 100% 규칙 | 상위 레벨의 모든 작업이 하위 레벨에 포함 |
| MECE 원칙 | 상호 배타적(Mutually Exclusive), 완전 포괄적(Collectively Exhaustive) |

```
                │         프로젝트 전체            │

                              │
  Level 1      ┌──────────────┼──────────────┐
               │              │              │
          ┌────┴────┐   ┌────┴────┐   ┌────┴────┐
          │ 단계 1  │   │ 단계 2  │   │ 단계 3  │
          └────┬────┘   └────┬────┘   └────┬────┘
               │              │              │
  Level 2   ┌──┴──┐       ┌──┴──┐       ┌──┴──┐
           ┌┴┐  ┌┴┐     ┌┴┐  ┌┴┐     ┌┴┐  ┌┴┐
           │A│  │B│     │C│  │D│     │E│  │F│
           └─┘  └─┘     └─┘  └─┘     └─┘  └─┘
                          │
  Level 3              ┌──┼──┐
                      ┌┴┐┌┴┐┌┴┐
                      │1││2││3│ ◀── Work Package
                      └─┘└─┘└─┘     (작업 패키지)

  │              = 100%                     │
  └────────────────────────────────────────┘
```
