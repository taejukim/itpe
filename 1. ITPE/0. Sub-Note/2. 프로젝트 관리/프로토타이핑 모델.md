## 정의
원활한 의사소통 개발 모델, 프로토타이핑 모델 개요
- 시스템의 주요 기능을 모형(Prototype)으로 만들어 사용자에게 먼저 보여주고 피드백을 수렴하는 방식
- 사용자 요구사항이 불명확한 경우 오해를 줄이고 시스템의 이해도를 높이기 위해 활용함
특징) PoC

## 목적
- 요구사항이 불명확한 초기 단계에서 실제 시스템의 일부 기능을 시각적으로 구현하여 고객의 요구를 조기에 명확화
- 고객 피드백을 신속하게 반영하여 요구사항의 오류를 줄이고 이해관계자 간의 의사소통 간극 해소
- 시스템 개발의 타당성을 검토하고 인터페이스의 사용성을 사전에 검증하여 최종 제품의 완성도 향상

## 구성요소
- 요구사항 분석: 기본적인 요구사항 및 목표 정의
- 프로토타입 설계: 빠른 시간 내에 핵심 기능 위주로 모형 설계
- 프로토타입 구축: 실제 동작이 가능한 수준으로 견본 제작
- 사용자 평가: 사용자가 프로토타입을 직접 사용하고 수정 사항 도출
- 정교화(Refinement): 평가 결과를 바탕으로 프로토타입 수정 및 보완
- 본 개발: 승인된 프로토타입을 바탕으로 전체 시스템 구축
