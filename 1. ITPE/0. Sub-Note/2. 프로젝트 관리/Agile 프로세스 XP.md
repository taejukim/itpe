## 정의
개발자 중심의 기술적 실천을 강조하는 애자일 방법론, XP
- 의사소통 개선, 즉각적인 피드백에 의해 단순하게 코딩하여 SW품질을 높이기 위한 방법론

## 5개의 가치
* 용기, 피드백, 의사소통, 단순함, 존중
* 
## 목적
-  비즈니스 가치 중심의 신속한 소프트웨어 인도
- 리팩토링과 지속적 통합(CI)을 통해 변화에 유연한 소프트웨어 구조를 유지하고 기술 부채 관리

## 구성요소
- 개발 : 페어프로그래밍, 공통책임, CI
- 관리 : Planning Process, Small Release, 메타포(소통)
- 구현 : Simple Design, TDD, 리팩토링
- 환경 : 40시간 근무, 고객의 상주
- 기타 : 코드 표준

## 구성도 요소
* 프로세스: User Story → Planning → Iteration (TDD/Pair) → Release → Feedback
* 관계: 개발 생산성과 품질을 동시에 높이기 위한 엔지니어링 중심 접근
```
[User Story] -> [구조적 Spike] -> [Release Plan] -> [Iteration]
                                                       |
                                              [TDD / Pair / CI]
                                                       |
                                            [Small Release] -> [Spike] 
                                                              ProtoType
```

## 키워드
* TDD, 페어 프로그래밍, 리팩토링, 지속적 통합, 5대 가치
