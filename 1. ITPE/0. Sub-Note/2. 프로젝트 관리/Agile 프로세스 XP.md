## 정의
Agile 프로세스 XP
고품질 소프트웨어를 빠르게 개발하기 위한 Agile 방법론, Extreme Programming
- 소프트웨어 품질을 향상시키고 고객의 요구사항에 빠르게 대응하는 개발 방법론
- 짧은 개발 주기와 지속적인 고객 피드백을 통한 점진적 개선을 강조

## 구성요소
| 실천법 | 영문 | 설명 |
|--------|------|------|
| 짝 프로그래밍 | Pair Programming | 두 명이 함께 코딩 |
| 테스트 주도 개발 | TDD | 테스트 먼저 작성 후 코드 작성 |
| 리팩토링 | Refactoring | 기능 유지하며 구조 개선 |
| 지속적 통합 | Continuous Integration | 코드를 자주 통합 |
| 간단한 설계 | Simple Design | 현재 요구사항에 맞는 최소 설계 |
| 집단 소유권 | Collective Ownership | 모든 코드는 모든 개발자가 수정 가능 |
| 코딩 표준 | Coding Standards | 일관된 코딩 규칙 |
| 고객 상주 | On-site Customer | 고객이 팀과 함께 근무 |

         |                        |                        |

                                  |

                    +---------------------------+
                    |        고객 상주          |
                    |   (즉각적 피드백 수집)    |
