## 정의
Agile 프로세스 Kanban(Development) 프로세스
작업 흐름을 시각화하고 지속적으로 개선하는 Agile 방법론, Kanban
- 작업을 시각적으로 표현하여 프로세스의 병목을 식별하고 개선하는 방법론
- 작업 진행 상태를 보드에 표시하여 팀의 작업 흐름을 투명하게 관리

## 목적
- 전체 작업 흐름을 시각화하여 병목 현상을 조기에 발견하고 프로세스의 흐름을 최적화
- WIP(Work In Progress) 제한을 통해 팀의 과부하를 방지하고 작업의 처리 속도(Throughput) 향상
- 투명한 작업 관리를 통해 팀원 간의 협업을 증진하고 지속적인 프로세스 개선 문화 정착

## 구성요소
- Kanban 보드: 작업을 시각적으로 표현하는 보드 (To Do, In Progress, Done 등)
- 작업 항목: 각 작업을 카드로 표현하여 보드에 배치
- WIP 제한: 진행 중인 작업(Work In Progress)의 수를 제한하여 병목 해소
- 풀 시스템: 다음 단계가 준비되면 이전 단계에서 작업을 가져오는 방식
- 지속적 개선: 프로세스를 지속적으로 모니터링하고 개선
- 흐름 최적화: 작업 흐름을 최적화하여 처리 시간 단축

## 구성도
To Do → In Progress → Review → Done (각 단계별 WIP 제한)

## 키워드
- Kanban 보드, WIP 제한, 풀 시스템, 지속적 개선, 흐름 최적화, [[Lean 경영 _ Lead SW Development]]

## 암기법
- 보작풀지흐: 보드, 작업 항목, 풀 시스템, 지속적 개선, 흐름 최적화
