## 정의
Agile 프로세스 Scrum
반복적이고 점진적인 소프트웨어 개발을 위한 Agile 프레임워크, Scrum
- 복잡한 제품 개발을 위한 경량급 프로세스 프레임워크
- 짧은 반복(Sprint)을 통해 지속적으로 가치를 전달하는 반복적 개발 방법론
- [[Agile 프로세스 선언문 4개문장]]과 [[Agile 프로세스 12개 원칙]]을 실현하는 대표적 방법론

## 목적
- 짧은 반복 주기(Sprint)를 통해 고객에게 지속적으로 가치를 전달하고 요구사항 변화에 유연하게 대응
- 명확한 역할(PO, SM, Team)과 이벤트(Scrum)를 통해 팀의 책임감을 높이고 협업 효율성 극대화
- 매일 진행 상황을 점검하고 회고를 통해 장애물을 신속히 제거하며 제품의 품질과 팀 성과를 지속적으로 개선

## 구성요소
| 구분 | 요소 | 설명 |
|------|------|------|
| 역할 | Product Owner | 제품 백로그 관리, 우선순위 결정 |
| | Scrum Master | 프로세스 촉진, 장애물 제거 |
| | 개발 팀 | 자기조직화된 팀, 실제 개발 수행 |
| 이벤트 | Sprint | 1~4주의 고정된 반복 주기 |
| | Sprint Planning | Sprint 목표와 작업 계획 수립 |
| | Daily Scrum | 매일 15분, 진행 상황 공유 |
| | Sprint Review | 결과물 검토 및 피드백 |
| | Sprint Retrospective | 프로세스 개선점 도출 |
| 산출물 | Product Backlog | 제품 요구사항 목록 |
| | Sprint Backlog | Sprint 수행 작업 목록 |
| | Increment | 완료된 제품 증분 |

## 암기법
- 제스개: 제품 책임자, Scrum Master, 개발 팀 (3가지 역할)
- 스계일검회: Sprint, 계획 회의, 일일 Scrum, 검토, 회고 (5가지 이벤트)
- 제스증: 제품 백로그, Sprint 백로그, 증분 (3가지 산출물)
