#프로젝트관리 #범위관리

## 정의
프로젝트 성공을 위해 수행해야 할 작업을 정의하고 통제하는 활동, 범위관리
- 프로젝트 완수를 위해 필요한 작업만을 포함하고, 필요하지 않은 작업은 제외함을 보장하는 프로세스
- 요구사항을 명확히 정의하고 [[WBS]]를 통해 상세화하여 작업의 누락과 중복 방지
## 키워드
* WBS, 범위 기준선, 요구사항 추적성 매트릭스(RTM), 범위 크리프(Scope Creep) 방지
## 암기법
* 계요정W확통
## 목적
- 프로젝트 목표 달성에 필요한 작업만을 포함하고 불필요한 작업(Scope Creep)을 배제하여 프로젝트의 본질과 자원 집중 보장
- 요구사항을 기반으로 범위를 명확히 정의하고 WBS를 통해 상세화함으로써 관리 가능한 수준의 작업 단위 확립
- 범위의 변경을 엄격히 통제하고 산출물의 인수 기준을 명확히 하여 이해관계자의 만족도 향상 및 분쟁 방지
## 구성요소
* 계획수립 : 범위관리 계획서
- 요구사항 수집: 이해관계자의 니즈와 요구사항을 파악하고 기록
- 범위 정의: 프로젝트와 인도물에 대한 상세한 설명서 작성 (Scope Statement)
- [[WBS]] 작성: 인도물 중심의 하위 작업 단위로 계층적 분할
- 범위 확인: 완성된 인도물을 이해관계자로부터 공식적으로 승인받는 과정
- 범위 통제: 범위 기준선에 대한 변경 사항을 모니터링하고 관리
## 구성도 요소
* 프로세스: 계획 수립 → 요구사항 수집 → 범위 정의 → WBS 작성 → 확인 → 통제
* 핵심 산출물: 범위 기준선(Scope Baseline = 범위 기술서 + WBS + WBS 사전)
```
[Requirements] -> [Scope Statement] -> [WBS] -> [Scope Baseline]
```
