## 정의
CI, CD
지속적 통합과 지속적 배포를 통한 소프트웨어 개발 자동화, CI/CD
- 코드 변경을 자동으로 통합, 테스트, 배포하여 개발 효율성을 향상시키는 방법론
- 수동 작업을 최소화하고 배포 주기를 단축하여 고품질 소프트웨어를 빠르게 전달
- [[DevOps]]의 핵심 구성요소로, [[Agile 프로세스 XP]]의 지속적 통합 개념을 확장

## 목적
- 코드 변경 사항의 빈번한 통합과 자동화된 빌드/테스트를 통해 결함을 조기에 발견하고 소프트웨어 품질 확보
- 반복적인 수동 배포 과정을 자동화하여 인적 오류를 방지하고 신속한 제품 출시(Time-to-Market) 실현
- 개발과 배포의 피드백 루프를 단축하여 사용자 요구사항에 민첩하게 대응하고 개발 생산성 극대화

## 구성요소
| 구분  | 용어                     | 설명                          |
| --- | ---------------------- | --------------------------- |
| CI  | Continuous Integration | 코드 변경을 자주 통합하고 자동 빌드/테스트 수행 |
| CD  | Continuous Delivery    | 언제든 배포 가능한 상태 유지 (수동 배포 승인) |
| CD  | Continuous Deployment  | 자동으로 프로덕션 환경에 배포 (완전 자동화)   |
|     |                        |                             |
