#AI #LLM #검색증강

## 정의
생성 과정 중 검색을 인터리브하는 기법, RIG
- Retrieval Interleaved Generation
- 텍스트 생성 과정에서 필요할 때마다 외부 지식을 검색하여 통합하는 기법
- RAG의 발전된 형태로, 생성 중간에 동적으로 검색 수행
## 키워드
* RIG, Interleaved, 동적 검색, RAG 발전, 적응적 검색
## 암기법
* RIG = Retrieval(검색) + Interleaved(인터리브) + Generation(생성)
* RAG: 검색 → 생성 / RIG: 검색 ↔ 생성 (반복)

## RAG vs RIG
| 구분 | RAG | RIG |
|------|-----|-----|
| 검색 시점 | 생성 전 1회 | 생성 중 여러 번 |
| 검색 방식 | 정적 | 동적 |
| 컨텍스트 | 고정 | 적응적 |
| 복잡도 | 낮음 | 높음 |
## 동작 원리
```
┌─────────────────────────────────────────────────┐
│              RIG 동작 과정                       │
│                                                  │
│  [질의] → [생성 시작]                           │
│              │                                  │
│              ▼                                  │
│         [토큰 생성]                             │
│              │                                  │
│         ┌────┴────┐                            │
│         │검색 필요?│                            │
│         └────┬────┘                            │
│         Yes  │  No                              │
│         ▼    │  │                              │
│     [검색]   │  │                              │
│         │    │  │                              │
│         ▼    ▼  ▼                              │
│     [결과 통합] → [계속 생성] → [완료]         │
└─────────────────────────────────────────────────┘
```
## 핵심 기술
| 기술 | 설명 |
|------|------|
| ==검색 트리거== | 검색이 필요한 시점 판단 |
| ==동적 쿼리 생성== | 현재 컨텍스트 기반 검색어 |
| ==결과 통합== | 검색 결과를 생성에 반영 |
| ==반복 검색== | 필요시 여러 번 검색 |
## 장점
| 장점 | 설명 |
|------|------|
| 정확성 향상 | 필요한 정보를 적시에 검색 |
| 최신 정보 | 생성 중 최신 데이터 반영 |
| 컨텍스트 효율 | 필요한 정보만 검색 |
| 복잡한 질의 | 다단계 추론 지원 |
## 과제
- 검색 시점 판단의 정확성
- 검색 지연으로 인한 속도 저하
- 검색 결과 품질 의존성
- 계산 비용 증가
