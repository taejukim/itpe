#AI #머신러닝 #클러스터링 #필수

## 정의
밀도 기반의 클러스터링 알고리즘, DBSCAN
- Density-Based Spatial Clustering of Applications with Noise
- 데이터 포인트의 밀도를 기반으로 클러스터를 형성하고, 노이즈를 자동으로 탐지하는 비지도학습 알고리즘
## 키워드
* DBSCAN, 밀도 기반, Epsilon, MinPts, 핵심점, 노이즈
## 암기법
* DBSCAN = Density(밀도) + Based(기반) + Clustering
* 핵심 파라미터: ==ε(반경), MinPts(최소 이웃)==
## 연관 토픽
- [[KNN]] - 이웃 기반 알고리즘
- [[PCA(주성분분석)]] - 차원 축소
- [[오토인코더(Autoencoder)]] - 이상 탐지
## 핵심 개념
| 개념                    | 설명                |
| --------------------- | ----------------- |
| ==ε (Epsilon)==       | 이웃 탐색 반경          |
| ==MinPts==            | 핵심점 판단 최소 이웃 수    |
| ==핵심점(Core Point)==   | ε 내에 MinPts 이상 이웃 |
| ==경계점(Border Point)== | 핵심점의 이웃이지만 핵심점 아님 |
| ==노이즈(Noise)==        | 어떤 클러스터에도 속하지 않음  |
|                       |                   |
## 동작 원리
```
┌─────────────────────────────────────────────────┐
│              DBSCAN 분류                         │
│                                                 │
│     ● ● ●                                       │
│   ● ★ ● ●     ★: 핵심점 (Core)                   │
│     ● ● ●     ●: 경계점 (Border)                 │
│                ○: 노이즈 (Noise)                  │
│           ○                                     │
│                                                 │
│  ε = 반경, MinPts = 4                            │
│  ★ 주변 ε 내에 4개 이상 → 핵심점                      │
└─────────────────────────────────────────────────┘
```
## 알고리즘 절차
1. 임의의 점 선택
2. ε 반경 내 이웃 탐색
3. MinPts 이상이면 핵심점, 클러스터 시작
4. 핵심점의 이웃들을 클러스터에 추가
5. 연결된 모든 핵심점 탐색
6. 남은 점들 중 미방문 점에 대해 반복
7. 어떤 클러스터에도 속하지 않으면 노이즈
## DBSCAN vs K-Means
| 구분 | DBSCAN | K-Means |
|------|--------|---------|
| 클러스터 수 | 자동 결정 | 사전 지정 |
| 클러스터 형태 | 임의 형태 | 구형 |
| 노이즈 처리 | 자동 탐지 | 없음 |
| 파라미터 | ε, MinPts | K |
## 장단점
| 장점 | 단점 |
|------|------|
| 클러스터 수 자동 결정 | ε, MinPts 선택 어려움 |
| 임의 형태 클러스터 | 밀도 차이에 민감 |
| 노이즈 자동 탐지 | 고차원에서 성능 저하 |
| 이상치에 강함 | 대용량 데이터 느림 |
