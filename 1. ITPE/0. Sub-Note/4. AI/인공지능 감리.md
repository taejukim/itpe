#AI #감리 #품질관리

## 정의
AI 시스템의 품질과 적합성을 검증하는 활동, 인공지능 감리
- AI 시스템의 개발, 운영 전 과정에서 품질, 윤리, 법적 준수 여부를 독립적으로 검증하고 평가하는 활동
- 신뢰할 수 있는 AI 구현을 위한 제3자 검증 체계
## 키워드
* AI 감리, 품질 검증, 윤리 감리, 기술 감리, 제3자 검증
## 암기법
* 감리 영역: ==기윤법== (기술, 윤리, 법적)
* 감리 단계: ==기데모테운== (기획, 데이터, 모델, 테스트, 운영)

## 감리 단계
| 단계 | 감리 내용 |
|------|-----------|
| ==기획== | 목표 적절성, 위험 분석, 윤리 검토 |
| ==데이터== | 데이터 품질, 편향, 개인정보 |
| ==모델 개발== | 알고리즘 적절성, 성능, 설명가능성 |
| ==테스트== | 검증, 공정성 테스트, 보안 |
| ==운영== | 모니터링, 드리프트, 사고 대응 |
## 감리 점검 항목
| 영역 | 점검 항목 |
|------|-----------|
| 데이터 | 출처, 품질, 편향, 라벨링 정확성 |
| 모델 | 정확도, 공정성, 견고성, 설명가능성 |
| 시스템 | 보안, 가용성, 성능, 확장성 |
| 프로세스 | 개발 방법론, 문서화, 변경 관리 |
| 거버넌스 | 정책, 조직, 역할, 책임 |
## AI 감리 vs 기존 IT 감리
| 구분 | 기존 IT 감리 | AI 감리 |
|------|-------------|---------|
| 대상 | 정보시스템 | AI 시스템 |
| 초점 | 기능, 성능 | 윤리, 공정성 추가 |
| 데이터 | 정합성 | 편향, 품질 |
| 산출물 | 명확 | 모델 (블랙박스) |
