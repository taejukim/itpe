#IT경영전략 #IT컴플라이언스

## 정의
IT 관점의 규제 준수를 위한 IT Compliance 정의
- 기업의 경영 투명성 제고 및 리스크 관리를 위해 **내부 정책과 외부 법적 규제 사항을 준수하는 체계**
- 데이터 보호, 재무 투명성, 정보보안 등과 관련된 의무적인 관리 활동 및 내부 통제 시스템
## 키워드
 - 리스크 관리, 내부 통제, 데이터 보존, 감사 추적(Audit Trail), ISMS-P, WORM, 
## 암기법
- **주요관점**: **공보보책통** (공개성, 보존성, 보호성, 책임성, 내부통제)
- 특징 : **프단구경**(프로젝트 개선, 단기수요폭발, 구축 범위, 경쟁령강화)
- **추진과정**: **식분대모** (규제식별, 영향분석, 대응방안, 모니터링)
* **개정보** (개인정보, 정보관리, 정보보호)

## IT 컴플라이언스 주요 관점
| 관점           | 설명                   | 핵심 내용                       |     |
| ------------ | -------------------- | --------------------------- | --- |
| ==**공개성**==  | 투명한 정보 공시 및 보고       | 재무 보고의 신뢰성, 공시 의무           |     |
| ==**보존성**==  | 법적 근거에 따른 데이터 장기 보관  | 이메일, 거래 로그, 증적 자료 보존        |     |
| ==**보호성**==  | 정보 자산의 기밀성, 무결성, 가용성 | 개인정보보호, 정보유출 방지             |     |
| ==**책임성**==  | 업무 내역 기록 및 사후 추적성    | 감사 추적(Audit Trail), R&R 명확화 |     |
| ==**내부통제**== | 내부 통제를 위한 규제준수       | 시스템적 규제 준수 대응               |     |
