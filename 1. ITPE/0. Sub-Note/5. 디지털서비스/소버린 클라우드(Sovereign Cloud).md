#정보관리기술사 #디지털서비스 #클라우드 #정리
## 정의
데이터 주권 보장 클라우드 아키텍처, 소버린 클라우드(Sovereign Cloud)
- 국가 및 특정 지역 법률·규정을 준수하고 데이터에 대한 주권을 생성 국가에 부여하는 클라우드 컴퓨팅 아키텍처
- 현지 법률과 규정을 준수하여 현지에 데이터센터를 두고, 데이터 접근이 이루어지도록 설계된 주권형 클라우드
## 키워드
* 운영주권, 데이터주권, 소프트웨어주권, 지역적 통제, 법률적 통제, 거주 통제, 신분 및 접근 제어, 규제 준수
## 암기법
* 운데소 (운영주권, 데이터주권, 소프트웨어주권) - 3대 주권
* 지법거신규 (지역적 통제, 법률적 통제, 거주 통제, 신분/접근 제어, 규제 준수) - 5대 통제 영역
## 특징
- 데이터 레지던시: 데이터가 특정 지역/국가 내에서만 저장·처리되도록 보장
- 주권 통제: 운영주권, 데이터주권, 소프트웨어주권의 3대 주권 보장
- 접근 제한: 국적, 물리적 위치 등에 따라 데이터 접근권한 제한
- 규제 준수: 각국 데이터 보호법(GDPR, 개인정보보호법 등) 준수 지원
## 목적
- 국가·지역별 데이터 주권 및 디지털 주권 확보
- 데이터 보호법(GDPR, 개인정보보호법 등) 규제 준수 보장
- AI/ML 등 데이터 집약적 기술 활용 시 데이터 통제권 유지
- 의료, 금융, 공공 등 규제 산업의 클라우드 전환 지원
## 구성요소
| 구분 | 구성요소 | 설명 |
|------|----------|------|
| 주권 | 데이터 주권(Data Sovereignty) | 데이터 저장·처리 위치의 법적 관할권 보장 |
| 주권 | 운영 주권(Operational Sovereignty) | 클라우드 운영·관리의 현지 통제 보장 |
| 주권 | 소프트웨어 주권(SW Sovereignty) | SW 소스코드·업데이트의 자주적 통제 |
| 통제 | 지역적 통제 | 데이터센터의 물리적 위치 제한 |
| 통제 | 법률적 통제 | 현지 법률·규정 준수 체계 |
| 통제 | 거주 통제(Data Residency) | 데이터의 국경 내 상주 보장 |
| 보안 | 신분 및 접근 제어 | 국적·위치 기반 접근권한 관리 |
| 보안 | 암호화·네트워크 세분화 | 국가별 보안 기준 준수 |
