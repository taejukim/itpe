## 정의
클라우드 구축용 오픈소스 플랫폼, 오픈스택
- 클라우드를 구축하기 위해 가상화된 자원을 통합, 제어, 운영하기 위한 오픈소스 플랫폼
- 파이썬 기반, 아파치 2.0 라이선스로 개방적 설계 및 개발 추구
## 키워드
* Nova, Glance, Neutron, Cinder, Swift, Keystone, Horizon, Heat, Ceilometer, RestAPI
## 암기법
- 서스사관D컨네(서버, 스토리지, 사용자, 관리, DB, 컨테이너, 네트워크)
* 호신매뉴히트 노바스글시키: Horizon, Cinder, Magnum, Neutron, Heat, Trove / Nova, Swift, Glance, Ceilometer, Keystone
## 특징
- RestAPI: 표준 REST API 기반의 서비스 제공
- HW 제한 없음: 특정 하드웨어에 종속되지 않는 유연한 구성
- 다양한 형태 구축: Public/Private/Hybrid 클라우드 모두 지원
- 열린 설계/개발: 파이썬 기반, 아파치 2.0 라이선스, 커뮤니티 주도
## 목적
- 벤더 Lock-in 방지를 통한 클라우드 독립성 확보
- 기능 개선 사항의 커뮤니티 공유를 통한 빠른 발전
- 클라우드 간 이동 편리성 확보(이식성, 상호운용성)
## 구성요소
| 구분 | 프로젝트 | 기능 |
|------|----------|------|
| 서버 | Nova | 컴퓨팅(VM 생성/관리) |
| 서버 | Glance | 이미지 서비스(VM 이미지 관리) |
| 스토리지 | Cinder | 블록 스토리지 |
| 스토리지 | Swift | 오브젝트 스토리지 |
| 사용자 | Horizon | 대시보드(Web I/F) |
| 사용자 | Keystone | 인증/권한 관리 |
| 관리 | Ceilometer | 미터링/모니터링 |
| 관리 | Heat | 오케스트레이션 |
| DB | Trove | DBaaS(DB 서비스) |
| 컨테이너 | Magnum | CaaS(컨테이너 서비스) |
| 네트워크 | Neutron | 네트워크 가상화(SDN) |
