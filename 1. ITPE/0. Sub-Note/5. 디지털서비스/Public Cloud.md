#정보관리기술사 #디지털서비스 #클라우드배포모델 #정리

## 정의
클라우드 배포 모델의 유형, Public/Private/Hybrid Cloud
- Public Cloud: 일반 사용자를 대상으로 인터넷을 통해 IT 자원을 공개적으로 제공하는 클라우드 서비스
- Private Cloud: 기업 내부 사용자만을 대상으로 전용 인프라를 구축하여 운영하는 클라우드 서비스
## 키워드
* Public Cloud, Private Cloud, Hybrid Cloud, Multi-tenancy, SDC, SDDC, TCO
## 암기법
* 통가통통 빠P T빠보생: (Hybrid 구성/장점) 통합인증, 가상화연동, 통합개발, 통합관리 / 빠른IT환경구축, PP유리, TCO절감, 빠른확장, 보안강화, 생산성증진
## 특징
- Public Cloud: 다중 테넌트, 종량제 과금, 빠른 확장성
- Private Cloud: 보안 강화, 커스터마이징, 규제 준수 용이
- Hybrid Cloud: Public/Private 장점 결합, 워크로드 분산
- Multi Cloud: 복수 CSP 활용, 벤더 종속 방지
## 목적
- 기업 환경에 최적화된 클라우드 배포 전략 수립
- 보안/비용/유연성 간 균형 잡힌 IT 인프라 운영
## 구성요소
| 구분 | Public Cloud | Private Cloud | Hybrid Cloud |
|------|-------------|---------------|--------------|
| 대상 | 일반 사용자 | 기업 내부 사용자 | 양쪽 혼합 |
| 보안 | 공유 보안 모델 | Firewall, DMZ | 통합인증, 보안강화 |
| 인프라 | 공유 인프라 | SDC, SDDC | 가상화 연동 |
| 관리 | CSP 관리 | 자체 관리 | 통합 관리 |
| 비용 | 종량제(OpEx) | 초기투자(CapEx) | TCO 절감 |
| 확장성 | 높음 | 제한적 | 빠른 확장 |
## 구성도

```
┌─────────────────── Hybrid Cloud ───────────────────┐
│                                                     │
│  ┌── Public Cloud ──┐    ┌── Private Cloud ──┐     │
│  │ [일반 사용자]     │    │ [기업 내부 사용자] │     │
│  │ [공유 인프라]     │◀──▶│ [전용 인프라]     │     │
│  │ [종량제 과금]     │    │ [SDC/SDDC]       │     │
│  └──────────────────┘    └──────────────────┘     │
│         │         통합인증/가상화연동         │       │
│         └────────── 통합관리 ──────────────┘       │
└─────────────────────────────────────────────────────┘
```
