## 정의
OS 레벨 경량 가상화 기술, 컨테이너
- 하이퍼바이저 없이 호스트 OS의 커널을 공유하여 애플리케이션과 실행 환경을 격리된 공간에 패키징하는 OS 레벨 경량 가상화 기술
- 클라우드 네이티브 인프라의 핵심 구성요소로 빠른 배포, 이식성, 자원 효율성 제공
## 키워드
* Docker, LXC, LXD, OCI, 커널 공유, Namespace, Cgroup, 이미지, 레지스트리
* 컨테이너 런타임, containerd, CRI-O, 오버레이 네트워크
## 암기법
* 컨테이너 핵심 3요소: 네씨이 (Namespace-Cgroup-Image)
* VM vs 컨테이너: "하이퍼바이저 vs 커널공유"
## 특징
- ==경량성==: Guest OS 불필요, MB 단위 이미지로 빠른 기동
- ==이식성==: 개발·테스트·운영 환경 동일하게 동작 (Build Once, Run Anywhere)
- ==격리성==: Namespace로 프로세스·네트워크·파일시스템 격리
- ==자원효율성==: Cgroup으로 CPU·메모리 자원 제한 및 공유
## 목적
- 애플리케이션 배포·확장·관리의 일관성 확보
- 인프라 자원 활용 극대화 및 운영 비용 절감
## 구성요소
- ==컨테이너 런타임==: 컨테이너 생성·실행 엔진 (Docker, containerd, CRI-O)
- ==컨테이너 이미지==: 애플리케이션과 의존성을 패키징한 불변 템플릿
- ==이미지 레지스트리==: 이미지 저장·배포 저장소 (Docker Hub, Harbor)
- ==Namespace==: 프로세스·네트워크·파일시스템 격리 메커니즘
- ==Cgroup==: CPU·메모리·I/O 자원 제한 및 모니터링
- ==오버레이 네트워크==: 컨테이너 간 통신을 위한 가상 네트워크
- ==볼륨(Volume)==: 영속적 데이터 저장을 위한 스토리지

## VM vs 컨테이너 비교
| 구분 | VM | 컨테이너 |
|------|-----|---------|
| 가상화 수준 | 하드웨어 | OS(프로세스) |
| Guest OS | 필요 | 불필요 |
| 크기 | GB 단위 | MB 단위 |
| 시작 시간 | 분 단위 | 초 단위 |
| 격리 수준 | 강함 | 상대적 약함 |
| 자원 효율 | 낮음 | 높음 |
| 대표 기술 | VMware, KVM | Docker, containerd |
## 컨테이너 실행 표준
- ==OCI(Open Container Initiative)==: 컨테이너 이미지·런타임 표준
- ==CRI(Container Runtime Interface)==: 쿠버네티스 컨테이너 런타임 인터페이스
