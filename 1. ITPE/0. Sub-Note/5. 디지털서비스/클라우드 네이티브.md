#정보관리기술사 #디지털서비스 #클라우드네이티브 #필수 #정리

## 정의
마이크로 서비스 기반 클라우드 아키텍처, 클라우드 네이티브
- 마이크로 서비스로 구현한 클라우드 서비스 라이프 사이클 관리 아키텍처
- 클라우드 컴퓨팅의 이점을 최대한 활용하여 애플리케이션을 구축, 배포, 운영하는 소프트웨어 접근 방식
## 키워드
* MSA, Container, CD, DevOps, Serverless, CNCF, 12-Factor App, 쿠버네티스
## 암기법
* 마컨C데서 (MSA, Container, CD, DevOps, Serverless)
* 조아인배 (조직: DevOps/Agile, 아키텍처: MSA/API, 인프라: 컨테이너/K8s, 배포: CI/CD)
## 특징
- 마이크로서비스: 독립 배포 가능한 소규모 서비스 단위로 분리
- 컨테이너 기반: Docker, Kubernetes를 활용한 경량 가상화
- 지속적 배포: CI/CD 파이프라인을 통한 자동화된 빌드/배포
- 탄력적 확장: 수요에 따른 자동 스케일링 및 자가 치유
## 구성요소
| 구성요소 | 핵심기술 | 설명 |
|----------|----------|------|
| MSA | API GW, Polyglot, Tracking, Discovery | 마이크로서비스 아키텍처 |
| Container | Docker, Kubernetes, DockerFile | 컨테이너 기반 경량 가상화 |
| CD | Ansible, Puppet, Chef, Spinnaker | 지속적 배포 자동화 |
| DevOps | Culture, Automation, Measure, Share | 개발/운영 협업 문화 |
| Serverless | FaaS, BaaS | 서버 관리 없는 이벤트 기반 실행 |
