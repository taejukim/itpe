#정보관리기술사 #디지털서비스 #클라우드 #AutoScale #기출 #정리

## 정의
클라우드 환경의 능동적 자원 할당, Auto Scale
- 클라우드 환경에서 서비스 부하에 따라 자동으로 인스턴스 및 자원을 조절하여 능동적으로 자원을 할당/회수하는 기술
- 121회/122회 기출문제
## 키워드
* Scale Up, Scale Out, Scale In, Auto Healing, 임계치, 모니터링, 탄력성
## 암기법
* 업아웃인힐 (Scale Up, Scale Out, Scale In, Auto Healing)
## 특징
- 탄력성(Elasticity): 트래픽 변화에 따라 자원을 자동으로 확장/축소
- 비용 최적화: 사용량 기반 과금으로 유휴 자원 비용 절감
- 고가용성: 장애 발생 시 자동 복구(Auto Healing) 정책 적용
- 자동화: 임계치 기반 트리거로 수동 개입 없이 자원 관리
## 구성요소
| 유형 | 방식 | 설명 |
|------|------|------|
| Scale Up | 수직 확장 | CPU/Memory 등 기존 서버 자원 추가(스펙 향상) |
| Scale Out | 수평 확장 | 추가적인 Instance 할당(서버 수 증가) |
| Scale In | 자원 회수 | 늘렸던 자원을 다시 회수하는 작업 |
| Auto Healing | 자동 복구 | 장애 인스턴스 감지 및 자동 복구/교체 정책 |
## 구성도
```
                  ┌─────────────┐
                  │  모니터링     │

                         │ 트리거
           ┌─────────────┼─────────────┐
           ▼             ▼             ▼
    ┌──────────┐  ┌──────────┐  ┌──────────┐
    │ Scale Up │  │Scale Out │  │ Scale In │
    │(수직확장)│  │(수평확장)│  │(자원회수)│
    │CPU/Mem↑  │  │Instance↑ │  │Instance↓ │
    └──────────┘  └──────────┘  └──────────┘
                         │
                  ┌──────▼──────┐
                  │ Auto Healing│
                  │ (자동 복구)  │
                  └─────────────┘
```
