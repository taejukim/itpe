#정보관리기술사 #디지털서비스 #합의알고리즘 #블록체인 #122회 #정리

## 정의
블록체인 거래 검증 합의 메커니즘, 합의 알고리즘
- 분산 네트워크 참여자 간 거래의 유효성을 검증하고, 단일 원장 상태에 대한 합의를 도출하는 알고리즘
- 중앙기관 없이 비잔틴 장애를 허용하면서 분산 원장의 일관성을 보장하는 프로토콜
## 키워드
* PoW, PoS, BFT, PoET, Bitcoin-NG, Sharding, 비잔틴, 마이닝, TPS
## 암기법
* 작지비경셔 (작업증명-지분증명-비잔틴내결함-경과시간증명-샤딩)
## 특징
- 탈중앙 신뢰: 중앙 권위 없이 분산 참여자 간 신뢰 확보
- 비잔틴 내결함: 악의적 노드가 존재해도 합의 도출 가능
- 에너지 효율성 진화: PoW → PoS → PoET 등 에너지 소비 절감 방향으로 발전
- TPS 향상: Bitcoin-NG, Sharding 등 처리 성능 개선 기법 등장
## 목적
- 분산 네트워크에서 이중지불(Double Spending) 방지 및 거래 무결성 보장
- 블록체인 확장성(Scalability) 및 처리 성능 향상
## 구성요소
| 구분 | 합의 알고리즘 | 설명 |
|------|---------------|------|
| 작업증명 | PoW(Proof of Work) | 연산 작업량으로 블록 생성 권한 부여, 높은 에너지 소비 |
| 지분증명 | PoS(Proof of Stake) | 보유 지분량에 비례하여 블록 생성 권한 부여 |
| 비잔틴 | BFT(Byzantine Fault Tolerance) | 악의적 노드 1/3 미만 시 합의 보장 |
| 경과시간 | PoET(Proof of Elapsed Time) | TEE 기반 랜덤 대기시간 경과로 블록 생성 |
| 성능향상 | Bitcoin-NG | TPS 향상 프로토콜, Leader 기반 마이크로블록 |
| 확장성 | Sharding | 네트워크를 샤드로 분할하여 병렬 처리 |
## 구성도
```
[합의 알고리즘 분류]

  PoW ──────── 연산 경쟁 ──────── 높은 보안, 낮은 TPS
   │
  PoS ──────── 지분 기반 ──────── 에너지 절감, 지분 집중 위험
   │
  BFT ──────── 투표 합의 ──────── 빠른 확정, 노드수 제한
   │
  PoET ─────── 대기시간 경쟁 ──── TEE 기반, 공정한 선출
   │
  Bitcoin-NG ─ Leader 선출 ────── Key Block + Micro Block
   │
  Sharding ─── 네트워크 분할 ──── 병렬 처리, 확장성 확보
```
