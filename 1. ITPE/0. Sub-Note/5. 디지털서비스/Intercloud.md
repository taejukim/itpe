#정보관리기술사 #디지털서비스 #인터클라우드 #정리

## 정의
클라우드 간 상호연동 체계, Intercloud
- 서로 다른 클라우드 서비스 간의 인터페이스, 네트워크 프로토콜, 상호호환성을 통해 연동하는 기술
- 멀티클라우드 환경에서 이기종 클라우드 간 자원 공유 및 서비스 연계를 가능하게 하는 개념
## 키워드
* 인터클라우드, 상호호환성, 네트워크 프로토콜, 멀티클라우드, 페더레이션, 브로커링
## 암기법
* 인프상: 인터페이스, 프로토콜, 상호호환성
## 특징
- 상호운용성: 이기종 클라우드 간 표준 인터페이스 기반 연동
- 네트워크 프로토콜: 클라우드 간 통신을 위한 표준 프로토콜 활용
- 페더레이션: 복수 클라우드 자원의 논리적 통합 운영
- 브로커링: 최적 클라우드 서비스 중개 및 자원 배분
## 목적
- 특정 CSP 벤더 종속(Lock-in) 방지
- 멀티클라우드 환경의 유연한 자원 활용 및 최적화
## 구성요소
| 구분 | 항목 | 설명 |
|------|------|------|
| 인터페이스 | 표준 API | 클라우드 간 서비스 호출을 위한 표준 인터페이스 |
| 네트워크 | 프로토콜 | 클라우드 간 통신/데이터 교환 프로토콜 |
| 연동 | 상호호환성 | 이기종 클라우드 간 데이터/서비스 호환 |
| 관리 | 페더레이션 | 복수 클라우드의 논리적 통합 관리 |
| 중개 | 브로커링 | 최적 서비스/자원 중개 및 배분 |
## 구성도
```
[Cloud A (CSP1)] ◀──── 표준 API/프로토콜 ────▶ [Cloud B (CSP2)]
       │                                              │
       └──────────── [Intercloud Layer] ──────────────┘
                     │  상호호환성  │
                     │  페더레이션  │
                     │  브로커링    │
                     └─────────────┘
                            │
                     [Cloud C (CSP3)]
```
