#정보관리기술사 #디지털서비스 #클라우드 #정리

## 정의
인터넷 기반 IT 자원 온디맨드 서비스, 클라우드 컴퓨팅
- 인터넷 기반으로 IT 자원(서버, 스토리지, 네트워크 등)을 온디맨드로 서비스 형태로 제공하는 컴퓨팅 모델
- 사용한 만큼 비용을 지불(Pay-Per-Use)하며, 탄력적 확장이 가능한 유틸리티 컴퓨팅
## 키워드
* On-Demand, Multi-tenants, Pay-Per-Use, 가상화, 프로비저닝, SLA, 유틸리티 컴퓨팅
## 암기법
* 온멀유서 (On-Demand, Multi-tenants, Utility Computing, 서비스)
- 서대관(멀)C분병: (6대 핵심기술) 서버가상화, 대용량클라우드, 가상서버관리(멀티테넌시), CDN, 분산스토리지, 병렬처리
- 오모빌네 보P브모: (9대 주변기술) 오픈API, 모니터링, 빌링, 네트워크가상화(VLan), 보안SLA, P2V/V2V, VPC, 모바일클라우드
- 사클마확서안: (전환절차) 사전분석, 클라우드전략, 마이그레이션, 확장, 서비스운영, 안정화
## 특징
- On-Demand: 필요 시 즉시 자원 할당 및 해제
- Multi-tenants: 다수 사용자가 동일 인프라를 공유
- Utility Computing: 사용량 기반 과금(Pay-Per-Use)
- 서비스 지향: IaaS, PaaS, SaaS 등 서비스 모델로 제공
## 목적
- IT 인프라 초기 투자 비용(CapEx) 절감 및 운영 비용(OpEx) 최적화
- 비즈니스 요구에 따른 탄력적 자원 확장/축소
- IT 자원 관리 부담 경감을 통한 핵심 역량 집중
## 구성요소
| 구분 | 구성요소 | 설명 |
|------|----------|------|
| 사용자 | 유저 인터랙션 I/F | 클라우드 서비스 접근 인터페이스 |
| 서비스 | 서비스 카탈로그 | 제공 가능한 서비스 목록 관리 |
| 관리 | System Management | 시스템 통합 관리 |
| 배포 | 프로비저닝 Tool | 자원 자동 할당/배포 |
| 자원 | On-Demand Self-Service | 셀프서비스 기반 자원 요청 |
| 운영 | 모니터링 & 미터링 | 자원 사용량 감시 및 측정 |
| 과금 | Pay-Per-Use | 사용량 기반 과금 체계 |
## 구성도

```
[사용자] ──▶ [유저 인터랙션 I/F]
                    │
            [서비스 카탈로그]
                    │
     ┌──────────────┼──────────────┐
     ▼              ▼              ▼
[IaaS]         [PaaS]         [SaaS]
(서버/스토리지)  (개발플랫폼)    (애플리케이션)
     │              │              │
     └──────────────┼──────────────┘
                    ▼
  [프로비저닝] ── [모니터링/미터링] ── [Pay-Per-Use]
                    │
            [가상화 인프라 풀]
```
