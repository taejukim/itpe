#정보관리기술사 #디지털서비스 #클라우드서비스모델

## 정의
모든 것을 서비스로 제공, XaaS
- Everything as a Service, 클라우드 기반으로 IT 자원을 인프라부터 애플리케이션까지 서비스 형태로 제공하는 모델
- IaaS, PaaS, SaaS를 기본으로 BaaS, DaaS, SecaaS 등으로 확장
## 키워드
* IaaS, PaaS, SaaS, BaaS, DaaS, SecaaS, On-Demand, 프로비저닝, Pay-per-Use
## 암기법
* 컴스콘 컨시시: (PaaS 구성) Compute, Storage, Controller, Connect, CDN, Security
- 클에풀 브게인: (DaaS 기술요소) 클라이언트, 에이전트, 풀링, 브로커, 게이트웨이, 인프라
## 특징
- 서비스 추상화: 인프라~애플리케이션 계층별 서비스 분리 제공
- 종량제 과금: Pay-per-Use 기반 비용 효율성
- 프로비저닝: 자동화된 자원 배포 및 관리
- 확장성: 수요에 따른 탄력적 자원 확장/축소
## 목적
- IT 자원의 서비스화를 통한 비즈니스 민첩성 확보
- 인프라 투자 비용 절감 및 운영 효율성 극대화
## 구성요소
| 서비스 | 정의 | 주요 구성 |
|--------|------|-----------|
| IaaS | 인프라를 서비스로 제공 | Compute, Storage, Network |
| PaaS | 앱 개발/테스트/구축 통합 플랫폼 | 컴스콘 컨시시(Compute, Storage, Controller, Connect, CDN, Security) |
| SaaS | SW를 서비스로 제공 | 프로비저닝, Pay-per-Use |
| BaaS | 모바일 앱 백엔드 서비스 | 데이터저장, 푸시, 사용자관리, 표준API |
| DaaS | 데스크탑 가상화(VDI) 서비스 | 클에풀 브게인(클라이언트, 에이전트, 풀링, 브로커, 게이트웨이, 인프라) |
| SecaaS | 보안 기능의 클라우드 서비스 | IAM, DLP, 암호화, 네트워크보안 |
## 구성도
```
[사용자/기업]
     │
     ▼
┌─────────────────────────────────────┐
│              SaaS                    │  ← 애플리케이션 (CRM, ERP, 메일)
├─────────────────────────────────────┤
│              PaaS                    │  ← 개발 플랫폼 (런타임, DB, 미들웨어)
├─────────────────────────────────────┤
│              IaaS                    │  ← 인프라 (서버, 스토리지, 네트워크)
└─────────────────────────────────────┘
     │          │          │
  [BaaS]     [DaaS]    [SecaaS]  ← 확장 서비스
```
