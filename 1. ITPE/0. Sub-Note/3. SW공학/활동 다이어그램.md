#SW공학 #UML #행위다이어그램

## 정의
업무 흐름과 제어 흐름, 활동 다이어그램
- 시스템의 업무 처리 순서, 분기, 병렬처리 등 제어 흐름(Control Flow)을 표현하는 UML 다이어그램
- 알고리즘, 비즈니스 프로세스, 워크플로우를 시각적으로 모델링하여 동적 행위 표현
## 키워드
- Activity, Action, Initial/Final Node
- Decision, Merge, Fork, Join, Swimlane
- Control Flow, Workflow, 병렬처리
- 순차, 분기, 병합, 동기화
## 암기법
- 시종활분포조스
- 노드 4가지: 시종결병 (시작-종료-결정-병합)
- 병렬 2개: 포조 (Fork-Join)
- 흐름: 순분병합 (순차-분기-병렬-합병)
## 목적
- 비즈니스 프로세스 및 워크플로우 시각화
- 알고리즘 로직과 제어 흐름의 명확한 표현
## 구성요소
| 요소 | 표기 | 설명 |
|-----|------|------|
| 시작 | ● | 시작점 |
| 종료 | ◎ | 종료점 |
| 활동 | ▭ | 수행 작업 |
| 분기 | ◇ | 조건 분기 |
| 포크 | ─┬─ | 병렬 시작 |
| 조인 | ─┴─ | 병렬 종료 |
| 스윔레인 | │ │ | 역할 구분 |
