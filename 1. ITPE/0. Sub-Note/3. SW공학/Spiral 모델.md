## 정의
Spiral 모델
소프트웨어의 기능을 나누어 점진적으로 개발하는 모델로, PDCA 주기를 시스템 개발에 적용, Spiral 모델
- [[폭포수 모델]]과 [[프로토타이핑 모델]]의 장점을 수용하고 위험 관리를 추가한 반복적 개발 모델
- 각 반복마다 위험을 식별하고 평가하여 다음 단계 진행 여부를 결정하는 위험 중심 개발 방법론
- 특징: 대규모 프로젝트 적합, 위험 관리 중심, 반복적·점진적 개발

## 목적
- 대규모 및 고위험 프로젝트에서 각 반복 단계마다 위험 분석을 수행하여 프로젝트의 불확실성을 체계적으로 관리하고 통제
- 폭포수 모델의 체계성과 프로토타이핑 모델의 유연성을 결합하여 점진적으로 완성도 높은 시스템 구축
- 고객 평가를 통한 지속적인 피드백 반영으로 최종 제품에 대한 고객 만족도를 높이고 성공 가능성 제고

## 구성요소
| 단계      | 설명                | 주요 활동           |
| ------- | ----------------- | --------------- |
| 계획 수립   | 목표, 대안, 제약조건 결정   | 요구사항 정의, 목표 설정  |
| 위험 분석   | 대안 평가, 위험 식별 및 해결 | 위험 분석, 프로토타입 개발 |
| 개발 및 검증 | 제품 개발             | 설계, 구현, 테스트     |
| 고객 평가   | 결과 평가, 다음 단계 계획   | 고객 검토, 다음 반복 계획 |
