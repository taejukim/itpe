## 정의
시스템 기능과 액터 관계, 유스케이스 다이어그램
- 시스템이 제공하는 기능(Use Case)과 외부 사용자(Actor)의 상호작용을 표현하는 UML 다이어그램
- 요구사항 분석 단계에서 시스템의 기능적 범위와 사용자 관점의 요구사항을 시각화

## 특징
- 사용자 중심: 외부 관점 표현
- 기능 중심: 시스템 기능 명시
- 간결성: 고수준 개요 제공
- 추적성: 요구사항 추적 가능

## 목적
- 사용자 요구사항의 시각적 표현 및 기능 범위 정의
- 이해관계자 간 요구사항 의사소통 촉진

## 구성요소
- Actor(액터): 시스템 외부의 사용자, 다른 시스템 (막대인간 아이콘)
- Use Case(유스케이스): 시스템이 제공하는 기능 단위 (타원)
- System Boundary(시스템 경계): 시스템 범위 (사각형)
- Relationship(관계): Include, Extend, Generalization
- Include(포함): 필수적으로 포함되는 유스케이스 (<<include>>)
- Extend(확장): 선택적으로 확장되는 유스케이스 (<<extend>>)
- Generalization(일반화): 액터/유스케이스 간 상속관계

## 구성도
```
┌─────────────────────────────────────────┐
│          System Boundary               │
│  ┌────────────────────────────────┐    │
│  │                                │    │
│  │    ┌─────────────┐             │    │
│  │    │ Use Case 1  │             │    │
│  │    └─────────────┘             │    │
│  │                                │    │
│  │    ┌─────────────┐             │    │
│  │    │ Use Case 2  │             │    │
│  │    └─────────────┘             │    │
│  │                                │    │
│  └────────────────────────────────┘    │
└─────────────────────────────────────────┘
         ▲
         │ interacts
         │
    ┌────┴────┐
    │  Actor  │
    └─────────┘

[예시: 온라인 쇼핑몰]

                ┌─────────────────────────┐
                │  온라인 쇼핑몰 시스템    │
                │                         │
                │   ┌──────────────┐      │
                │   │  상품 검색    │      │
                │   └──────────────┘      │
                │          │              │
                │   <<include>>           │
  ┌──────┐      │          ▼              │
  │ 고객  │──────┤   ┌──────────────┐      │
  └──────┘      │   │  상품 구매    │      │
                │   └──────┬───────┘      │
                │          │              │
                │   <<extend>>            │
                │          ▼              │
                │   ┌──────────────┐      │
                │   │  할인 적용    │      │
                │   └──────────────┘      │
                │                         │
                │   ┌──────────────┐      │
  ┌──────┐      │   │  주문 관리    │      │
  │관리자 │──────┤   └──────────────┘      │
  └──────┘      │                         │
                └─────────────────────────┘

[관계 표기]
- Include: UseCase1 ──<<include>>──> UseCase2
- Extend:  UseCase1 <──<<extend>>── UseCase2
- Generalization: Actor1 △─── Actor2
```

## 키워드
- Actor, Use Case, System Boundary
- Include, Extend, Generalization
- 요구사항 분석, 기능 명세, 사용자 관점
- Association, 상호작용

## 암기법
- 유스케이스 3요소: 액유시 (Actor-UseCase-System)
- 관계 3가지: 인익제 (Include-Extend-Generalization)
- Include(필수) vs Extend(선택)
