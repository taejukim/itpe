#SW공학 #UML #행위다이어그램 #필수

## 정의
시스템 기능과 액터 관계, 유스케이스 다이어그램
- 시스템이 제공하는 기능(Use Case)과 외부 사용자(Actor)의 상호작용을 표현하는 UML 다이어그램
- 요구사항 분석 단계에서 시스템의 기능적 범위와 사용자 관점의 요구사항을 시각화
## 키워드
- Actor, Use Case, System Boundary
- Include, Extend, Generalization
- 요구사항 분석, 기능 명세, 사용자 관점
- Association, 상호작용
## 암기법
- 유스케이스 3요소: 액유시 (Actor-UseCase-System)
- 관계 3가지: 인익제 (Include-Extend-Generalization)
- Include(필수) vs Extend(선택)
## 목적
- 사용자 요구사항의 시각적 표현 및 기능 범위 정의
- 이해관계자 간 요구사항 의사소통 촉진
## 구성요소
| 요소 | 표기 | 설명 |
|-----|------|------|
| Actor | 🧑 | 시스템 외부 사용자/시스템 |
| Use Case | ⬭ | 시스템 기능 단위 |
| 시스템 경계 | □ | 시스템 범위 |
