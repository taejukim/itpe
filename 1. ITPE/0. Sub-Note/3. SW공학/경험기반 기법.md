## 정의
직관과 경험 기반 테스트, 경험기반 기법
- 테스터의 지식, 경험, 직관을 활용하여 결함을 찾아내는 비형식적 테스트 기법
- 명세나 구조 없이도 과거 경험과 도메인 지식으로 효과적인 테스트 수행

## 특징
- 직관성: 테스터 경험 활용
- 유연성: 형식에 구애받지 않음
- 창의성: 독특한 시나리오 발굴
- 효율성: 신속한 결함 발견

## 목적
- 테스터의 전문성과 직관을 활용한 효과적 결함 발견
- 형식적 기법으로 발견 어려운 결함 탐색

## 구성요소
- Exploratory Testing(탐색적 테스트): 동시에 학습-설계-실행
- Error Guessing(오류 추정): 과거 경험으로 결함 예측
- Ad-hoc Testing(임시 테스트): 사전 계획 없이 즉흥적 테스트
- Attack Testing(공격 테스트): 취약점 공격 시나리오 테스트
- Session-based Testing(세션 기반): 시간제한 내 집중 탐색
- Checklist-based Testing(체크리스트): 경험 기반 체크리스트 활용
- Defect Seeding(결함 주입): 고의적 결함 삽입 후 발견율 측정

## 구성도
```
[경험기반 테스트 특징]

┌────────────────────────────────┐
│  Formal Testing (형식적)        │
│  - 명세 기반 (블랙박스)          │
│  - 구조 기반 (화이트박스)        │
│  - 계획적, 체계적              │
└────────────────────────────────┘
               vs
┌────────────────────────────────┐
│  Experience-based (경험기반)    │
│  - 테스터 직관                  │
│  - 도메인 지식                  │
│  - 즉흥적, 유연함               │
└────────────────────────────────┘

[탐색적 테스트 프로세스]

┌─────────────┐
│   Charter   │ (테스트 목표 설정)
└──────┬──────┘
       │
       ▼
┌──────────────────────────────┐
│  Time-box Session (90분)     │
│  ┌────────────────────────┐  │
│  │  Learn (학습)          │  │
│  │       ↓                │  │
│  │  Design (설계)         │  │
│  │       ↓                │  │
│  │  Execute (실행)        │  │
│  │       ↓                │  │
│  │  Analyze (분석)        │  │
│  └────────────────────────┘  │
└──────────────────────────────┘
       │
       ▼
┌─────────────┐
│  Debrief    │ (결과 공유)
└─────────────┘

[오류 추정 예시]
- NULL 포인터 처리
- 경계값 오류 (0, -1, MAX)
- 동시성 문제 (Race Condition)
- 메모리 누수
- 정수 오버플로우
- SQL 인젝션
```

## 키워드
- Exploratory Testing, Error Guessing, Ad-hoc
- Session-based, Attack Testing, Checklist
- 테스터 직관, 도메인 지식, 유연성
- 비형식적, 즉흥적, 창의적

## 암기법
- 경험 3대 기법: 탐오임 (탐색적-오류추정-임시)
- 탐색적 4단계: 학설실분 (학습-설계-실행-분석)
- 특징: 테스터의 머리와 손에 의존
