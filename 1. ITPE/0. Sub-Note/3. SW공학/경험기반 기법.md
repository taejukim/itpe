## 주요 기법
##### 1. 탐색적 테스팅 (Exploratory Testing)
- **정의**: 테스트 설계와 실행을 동시에 수행
- **특징**: 절차 구애 없음, 기능 중심 신속 테스트
- **구성요소**:
  - Test Charter: 테스트 승인서, 목적/범위 명시
  - Time Boxing: 테스트 수행 시간 지정 (60/90/120분)
  - Session Sheet: 테스트 결과지, 아이디어/결과/결함 기록
  - Debriefing: 요약 보고, 경험 공유
- **단계**: 차터 수립 → 탐색/학습 → 설계/실행 → 기록/분석
##### 2. 오류 추정 (Error Guessing)
- **정의**: 과거 경험을 바탕으로 오류 발생 가능 부분 추정
- **특징**: 테스터의 직관과 경험 활용
- **방법**: 유사 어플리케이션, 기술 경험 활용
##### 3. 체크리스트 기반 (Checklist-based Testing)
- **정의**: 사전 정의된 체크리스트로 테스트
- **특징**: 체계적 접근, 누락 방지
##### 4. 분류 트리 (Classification Tree)
- **정의**: 입력 파라미터를 트리 구조로 분류
- **특징**: 체계적 테스트 케이스 도출
