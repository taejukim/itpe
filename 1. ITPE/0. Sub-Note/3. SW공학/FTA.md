## 정의
하향식 결함 분석 기법, FTA
- 시스템의 최상위 고장(Top Event)으로부터 원인을 하향식으로 추적하여 트리 구조로 분석
- 논리 게이트(AND, OR)를 사용하여 고장 원인의 조합과 확률을 정량적으로 분석

## 특징
- 연역적: 하향식(Top-Down) 접근
- 정량화: 확률 계산 가능
- 시각화: 트리 구조 표현
- 체계성: 논리 게이트 활용

## 목적
- 시스템 고장의 근본 원인 규명
- 고장 확률 분석 및 예방 대책 수립

## 구성요소
- Top Event(정상 사건): 분석 대상인 최상위 고장
- Basic Event(기본 사건): 더 이상 분해되지 않는 최하위 원인
- Intermediate Event(중간 사건): 상위와 하위를 연결하는 중간 사건
- AND Gate(논리곱): 모든 입력 발생 시 출력 발생
- OR Gate(논리합): 하나 이상의 입력 발생 시 출력 발생
- Cut Set(최소 절단 집합): 고장을 유발하는 기본 사건의 최소 조합
- Fault Tree Diagram(결함 트리 다이어그램): 트리 구조 시각화
- Probability Analysis(확률 분석): 각 사건의 발생 확률 계산

## 구성도
```
[FTA 기본 구조]

        ┌─────────────┐
        │ Top Event   │ ← 시스템 고장
        │ (정상 사건)  │
        └──────┬──────┘
               │
          ┌────┴────┐
          │   OR    │ ← 논리 게이트
          └────┬────┘
           ┌───┴───┐
           ▼       ▼
   ┌─────────┐ ┌─────────┐
   │Event A  │ │Event B  │ ← 중간 사건
   └────┬────┘ └────┬────┘
        │           │
     ┌──┴──┐     ┌─┴─┐
     │ AND │     │AND│
     └──┬──┘     └─┬─┘
    ┌───┴───┐   ┌──┴──┐
    ▼       ▼   ▼     ▼
  [BE1]  [BE2] [BE3] [BE4] ← 기본 사건

[논리 게이트 기호]

AND Gate:  ┌─┐
          └─┬─┘  (모든 입력 필요)
            │

OR Gate:   ┌─┐
          └─▽─┘  (하나 이상 입력)
            │

[확률 계산 예시]

시스템 고장 (Top Event)
    OR
   ┌─┴─┐
  A    B
 AND  AND
 ┌┴┐  ┌┴┐
 E1 E2 E3 E4

P(E1)=0.1, P(E2)=0.2, P(E3)=0.15, P(E4)=0.25

P(A) = P(E1) × P(E2) = 0.1 × 0.2 = 0.02
P(B) = P(E3) × P(E4) = 0.15 × 0.25 = 0.0375

P(Top) = 1 - (1-P(A)) × (1-P(B))
       = 1 - (0.98) × (0.9625)
       = 1 - 0.94325
       = 0.05675 (5.675%)

[최소 절단 집합]
- {E1, E2}
- {E3, E4}
```

## 키워드
- Top Event, Basic Event, Intermediate Event
- AND Gate, OR Gate, Cut Set
- 하향식 분석, 연역적 방법, 정량적 분석
- 항공, 원자력, 안전필수시스템
