#SW공학 #테스트 #품질측정 #필수

## 정의
테스트 완전성 측정 지표, 테스트 커버리지
- 테스트가 소프트웨어의 어느 정도를 검증했는지 정량적으로 측정하는 지표
- 코드, 기능, 요구사항 등의 테스트 수행 정도를 백분율로 표현하여 테스트 품질 평가
## 키워드
- Statement, Decision, Condition, MC/DC
## 암기법
- **강도 순서**: 문분조조다경
- **MC/DC**: 항공 SW 필수

## 커버리지 종류 (낮음→높음)
| 커버리지        | 설명           | 기준          |
| ----------- | ------------ | ----------- |
| 구문(SC)      | 모든 문장 1회 이상  | 실행 문장/전체 문장 |
| 분기(DC)      | 모든 분기 1회 이상  | T/F 모두 실행   |
| 조건(CC)      | 모든 조건 T/F    | 개별 조건       |
| 조건/분기(C/DC) | 조건+분기 결합     | 조건+분기       |
| MC/DC       | 각 조건이 결과에 영향 | 항공/의료       |
| 다중조건(MCC)   | 모든 조건 조합     | 2^n 조합      |
## 커버리지 강도
```
SC < DC < CC < C/DC < MC/DC < MCC
```
