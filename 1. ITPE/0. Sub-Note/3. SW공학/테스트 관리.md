## 정의
테스트 활동 총괄 관리, 테스트 관리
- 테스트의 계획, 통제, 분석, 설계, 구현, 실행, 완료의 전 과정을 체계적으로 관리하는 활동
- 테스트 자원, 일정, 위험, 결함을 효과적으로 관리하여 품질 목표 달성

## 특징
- 계획성: 체계적 테스트 계획 수립
- 통제성: 진행 상황 모니터링 및 통제
- 측정성: 메트릭 기반 관리
- 추적성: 결함 및 이력 추적

## 목적
- 테스트 프로세스의 효과적 관리 및 품질 목표 달성
- 테스트 자원 최적화 및 결함 관리 체계 구축

## 구성요소
- Test Strategy(테스트 전략): 조직 수준의 테스트 접근법
- Test Policy(테스트 정책): 테스트 원칙과 목표
- Test Plan(테스트 계획): 프로젝트별 테스트 계획서
- Test Estimation(테스트 추정): 공수, 일정, 비용 산정
- Test Monitoring(테스트 모니터링): 진행 상황 추적
- Test Control(테스트 통제): 편차 발생 시 조치
- Defect Management(결함 관리): 결함 추적 및 해결
- Test Metrics(테스트 메트릭): 측정 지표 (커버리지, 결함 밀도)
- Test Tool(테스트 도구): 자동화 도구, 관리 도구
- Configuration Management(형상 관리): 테스트 산출물 관리

## 구성도
```
[테스트 관리 프로세스]

┌─────────────────┐
│ Test Strategy   │ ← 조직 수준
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Test Policy    │ ← 원칙/목표
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   Test Plan     │ ← 프로젝트별 계획
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼
┌────────┐ ┌─────────┐
│Monitor │ │ Control │ ← 모니터링/통제
└────────┘ └─────────┘
    │         │
    └────┬────┘
         ▼
┌─────────────────┐
│  Test Report    │ ← 결과 보고
└─────────────────┘

[결함 관리 프로세스]

결함 발견 → 결함 등록 → 결함 분석 → 결함 할당
    ↓
결함 수정 → 결함 검증 → 결함 종료 → 결함 리포트

[테스트 메트릭]
- 결함 밀도 = 결함 수 / KLOC
- 테스트 커버리지 = 실행 항목 / 전체 항목
- 결함 제거율 = 발견 결함 / 전체 결함
- 테스트 진행률 = 완료 케이스 / 전체 케이스
```

## 키워드
- Test Strategy, Test Plan, Test Policy
- Test Monitoring, Test Control, Defect Management
- Test Metrics, Configuration Management
- JIRA, TestRail, Bugzilla

## 암기법
- 관리 핵심: 전계추통 (전략-계획-추정-통제)
- 메트릭 4가지: 밀커제진 (밀도-커버리지-제거율-진행률)
- 결함 생명주기: 발등분할수검종리 (발견-등록-분석-할당-수정-검증-종료-리포트)
