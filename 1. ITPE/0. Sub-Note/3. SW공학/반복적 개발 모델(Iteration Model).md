## 정의
반복적 개발 모델(Iteration Model)
사용자의 요구사항이나 개발 범위를 분해하여 점증적으로 반복 개발하여 최종 시스템으로 완성하는 개발 모델, 반복적 개발 모델
- 전체 시스템을 작은 단위로 나누어 반복적으로 개발하여 점진적으로 완성하는 방법론
- 각 반복마다 완전한 기능을 구현하여 점진적으로 시스템을 확장

## 목적
- 전체 시스템을 관리 가능한 작은 단위로 나누어 반복 개발함으로써 요구사항 변화에 유연하게 대응하고 리스크 조기 식별
- 각 반복 주기를 통해 실제 작동하는 소프트웨어 증분(Increment)을 인도하여 고객 가치를 지속적으로 창출
- 개발 과정의 불확실성을 줄이고 초기 단계의 오류를 반복 주기를 통해 지속적으로 수정 보완하여 품질 향상

## 구성요소
- 요구사항 분해: 전체 요구사항을 작은 단위로 분해
- 반복 계획: 각 반복에서 개발할 기능 정의
- 설계 및 개발: 반복 단위의 설계 및 개발
- 테스트: 개발된 기능의 테스트
- 통합: 각 반복 결과를 통합하여 시스템 구축
- 검증: 통합된 시스템 검증 및 다음 반복 계획

## 구성도
요구사항 분해 → 반복 계획 → 설계/개발 → 테스트 → 통합 → 검증 → 다음 반복
