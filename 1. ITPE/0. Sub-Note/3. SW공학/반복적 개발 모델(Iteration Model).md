## 정의
반복적 개발 모델(Iteration Model)
여러번 나누어 개발하는 반복적 개발 모델의 정의
- 일정한 시간 단위==(Iteration)==로 분석-설계-구현-테스트를 반복하여 완성도를 높여가는 모델
- 초기 요구사항이 불완전하더라도 반복을 통해 점진적으로 요구사항을 명확히 함

## 구성요소
- 이터레이션(Iteration): 반복되는 짧은 개발 주기 (통상 2~4주)
- 점진적 발전: 각 주기마다 새로운 기능을 추가하거나 기존 기능을 보완
- 사용자 피드백: 주기 종료 시점마다 결과물을 확인하고 다음 주기에 반영
- 위험 분산: 주기별로 테스트를 수행하여 프로젝트 초기에 결함이나 위험 발견
- 통합 및 릴리스: 각 주기의 결과물을 통합하여 실행 가능한 버전 유지

## 구성도 요소
* 프로세스: 수평적인 개발 단계가 아닌, 순환 구조를 가진 작은 폭포수들의 연속
* 관계: 애자일 방법론의 근간이 되는 개발 철학
```
[Init] -> [Iter 1 (A/D/C/T)] -> [Iter 2 (A/D/C/T)] -> [Release]
               ^                  |
               +--- (Feedback) ---+
```

## 키워드
