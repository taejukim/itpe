#SW공학 #품질모델 #표준 #필수

## 정의
소프트웨어 품질 특성 국제 표준, ISO/IEC 25010
- 소프트웨어 제품의 품질 특성과 평가 모델을 정의한 국제 표준 (ISO 9126의 후속)
- 사용 품질(Quality in Use)과 제품 품질(Product Quality)의 2가지 품질 모델 제시
## 키워드
- 25010, 제품 품질 9특성, 사용 품질 5특성
## 암기법
- **9특성**: 기실호 상신보 유연안 (기능-실행-호환-상호작용-신뢰-보안-유지보수-유연성-안정성)
- 사용 품질 5대: 효효만위범
## 목적
- 소프트웨어 품질 특성의 체계적 분류 및 평가 기준 제공
- 제품 및 사용 품질의 측정 및 개선 지원

## 특징
- 포괄성: 8대 품질 특성 정의
- 계층성: 특성-부특성 구조
- 표준화: 국제 표준 품질 모델
- 측정성: 평가 가능한 품질 정의
## 제품 품질 8대 특성
| 특성         | 설명                       |
| ---------- | ------------------------ |
| 기능 적합성     | 기능 완전성, 정확성              |
| 성능(실행) 효율성 | 시간/자원 효율성                |
| 호환성        | 공존성, 상호운용성               |
| 상호작용 능력    | 학습/운용 용이성                |
| 신뢰성        | 성숙성, 가용성, 복구성            |
| 보안성        | 기밀성, 무결성, 부인방지           |
| 유지보수성      | 분석/수정/테스트 용이성            |
| 유연성        | 적응성, 확장성, 설치성, 대체성       |
| 안정성        | 운영제약, 위험식별, 실패 안전, 위험 경고 |
## 사용 품질 5대 특성
| 특성     | 설명        |
| ------ | --------- |
| 효과성    | 목표 달성 정확성 |
| 효율성    | 자원 대비 성과  |
| 만족성    | 사용자 만족    |
| 위험 회피성 | 위험 완화     |
| 범위 포괄성 | 상황 대응     |
