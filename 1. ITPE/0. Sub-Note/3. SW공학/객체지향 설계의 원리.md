## 정의
객체지향 설계의 원리
객체지향 소프트웨어 설계 시 지켜야 할 핵심 원칙들, SOLID 원칙
- Robert C. Martin이 정의한 객체지향 설계의 5대 원칙
- 유지보수성, 확장성, 재사용성을 높이기 위한 설계 가이드라인

## 목적
- ==SOLID 원칙(SRP, OCP, LSP, ISP, DIP)==을 통해 변화에 유연하고 유지보수가 용이한 소프트웨어 구조 설계 지침 제공
- 클래스 간의 결합도는 낮추고 응집도는 높여 코드의 재사용성을 극대화하고 예기치 않은 부작용(Side Effect) 방지
- 요구사항 변경 시 기존 코드의 수정을 최소화하고 확장이 용이한 고품질 아키텍처 구축

## 구성요소 (SOLID)
| 원칙 | 영문명 | 설명 |
|------|--------|------|
| S | Single Responsibility | 클래스는 하나의 책임만 가져야 함 |
| O | Open-Closed | 확장에는 열려있고, 수정에는 닫혀있어야 함 |
| L | Liskov Substitution | 자식 클래스는 부모 클래스를 대체할 수 있어야 함 |
| I | Interface Segregation | 클라이언트가 사용하지 않는 인터페이스에 의존하지 않아야 함 |
| D | Dependency Inversion | 고수준 모듈은 저수준 모듈에 의존하지 않고, 추상에 의존해야 함 |

## 구성도
```
  ┌─────────────────────────────────────────────────────────────┐
  │                    SOLID 원칙                               │
  └─────────────────────────────────────────────────────────────┘
  
  [S] 단일책임원칙 (SRP)
  ┌──────────────┐          ┌──────────────┐  ┌──────────────┐
  │ 클래스 A       │    →     │ 클래스 A     │  │ 클래스 B     │
    │ 책임1+책임2   │          │ 책임1        │  │ 책임2        │
  └──────────────┘          └──────────────┘  └──────────────┘
  
  [O] 개방폐쇄원칙 (OCP)
  ┌──────────────┐
  │   기존 코드   │ ◀── 수정 금지 (Close)
  └──────┬───────┘
         │ 상속/구현
         ▼
  ┌──────────────┐
  │   확장 코드   │ ◀── 확장 허용 (Open)
  └──────────────┘
  
  [L] 리스코프 치환원칙 (LSP)
  ┌──────────────┐
  │   부모 클래스 │ ◀── 사용처
  └──────┬───────┘
         │ 대체 가능
         ▼
  ┌──────────────┐
  │   자식 클래스 │
  └──────────────┘
  
  [I] 인터페이스 분리원칙 (ISP)
  ┌──────────────────┐      ┌────────┐  ┌────────┐  ┌────────┐
  │ 비대한 인터페이스│  →        │ IF A   │  │ IF B   │  │ IF C   │
  └──────────────────┘      └────────┘  └────────┘  └────────┘
  
  [D] 의존성 역전원칙 (DIP)
  ┌──────────────┐     ┌──────────────┐
  │ 고수준 모듈     │     │ 고수준 모듈     │
  │      ↓       │  →  │      ↓       │
  │ 저수준 모듈     │     │   추상화      │
  └──────────────┘     │      ↑       │
                       │ 저수준 모듈     │
                       └──────────────┘
```
