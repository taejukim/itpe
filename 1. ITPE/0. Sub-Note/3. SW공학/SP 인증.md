#SW공학 #프로세스인증 #품질인증

## 정의
소프트웨어 프로세스 품질 인증, SP 인증
- 소프트웨어 개발 조직의 프로세스 능력과 품질 수준을 평가하여 등급을 부여하는 국내 인증 제도
- 한국정보통신기술협회(TTA)가 운영하며, CMMI 기반으로 프로세스 성숙도를 1~5등급으로 평가
## 키워드
- SP 인증, SW진흥법, NIPA, 5개 영역
## 암기법
- **5영역**: 프개지조개 (프로젝트-개발-지원-조직-프로세스 개선)
## 목적
- 소프트웨어 조직의 프로세스 능력 평가 및 인증
- 조직의 체계적 프로세스 개선 및 경쟁력 강화
## 연관 토픽
- [[CMMI]] - [[정보시스템 감리]]
## 특징
- 한국형 CMMI
- 5개 영역, 17개 평가항목
- 등급제: 1~5등급
## 5개 영역
| 영역      | 내용           |
| ------- | ------------ |
| 프로젝트 관리 | 계획, 모니터링     |
| 개발      | 요구사항, 설계, 구현 |
| 지원      | 형상관리, 품질보증   |
| 프로세스 관리 | 조직 프로세스      |
| 조직 관리   | 프로세스 개선      |
## 인증 등급
| 등급 | 이름 |
|-----|------|
| 2등급 | 관리 (계획-설계-고객요구-품질보증) |
| 3등급 | 정의 (측정-문제해결-조직프로세스) |
