#SW공학 #객체지향 #필수

## 정의
복잡한 실체에서 공통적인 핵심 특징만을 추출하여 단순화하는 기법, 추상화
- 불필요한 세부 사항은 제거하고 시스템의 본질적인 속성과 동작에 집중하는 프로세스
- 복잡도를 관리하고 재사용 가능한 모델을 설계하기 위한 필수적인 객체지향 원리
## 키워드
* 핵심 특징 추출, 복잡성 관리, 추상 클래스, 인터페이스, 모델링
## 암기법
* 복심추 (복잡한 것에서 핵심만 추출)
## 구성요소
- 데이터 추상화: 개체의 핵심 속성들을 추출하여 데이터 타입으로 정의
- 제어 추상화: 복잡한 로직을 메서드로 묶어 이름만으로 기능을 호출할 수 있게 함
- 클래스 추상화: 공통된 특징을 가진 객체들을 대표하는 클래스 정의
- 추상 클래스(Abstract Class): 하위 클래스에서 반드시 구현해야 할 인터페이스 제공
- 인터페이스(Interface): 극단적인 추상화 형태로, 행위의 명세만을 정의
## 구성도 요소
* 아키텍처: 실세계 개체 -> 핵심 특징 추출 -> 추상 모델(Class) 생성
* 관계: 추상화 수준이 높을수록 범용성이 증가하고 구체적인 구현(Implementation)에서 자유로워짐
```
[현실 세계의 차] --- (추상화) ---> [Car Class]
 (엔진, 타이어, ...)             (+ 달린다(), + 멈춘다())
```
