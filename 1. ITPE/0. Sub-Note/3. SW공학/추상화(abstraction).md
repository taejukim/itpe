## 정의
추상화(abstraction)
복잡한 시스템을 단순화하여 핵심적인 특징만 표현하는 객체지향 원칙, 추상화
- 불필요한 세부사항을 숨기고 필요한 정보만 노출하여 시스템의 복잡성을 관리하는 기법
- 추상 클래스나 인터페이스를 통해 공통된 특성을 정의하고 구체적인 구현은 하위 클래스에서 담당

## 구성요소
- 추상 클래스: 공통된 특성을 정의하지만 완전한 구현은 하지 않는 클래스
- 인터페이스: 메서드 시그니처만 정의하고 구현은 하지 않는 계약
- 구체 클래스: 추상 클래스나 인터페이스를 구현하는 실제 클래스
- 공통 특성: 여러 클래스가 공유하는 공통된 속성과 메서드
- 세부사항 은닉: 구현 세부사항을 숨기고 인터페이스만 노출

## 구성도
복잡한 시스템 → 추상화 → 추상 클래스/인터페이스 → 구체 클래스 구현

## 키워드
- 추상 클래스, 인터페이스, 구체 클래스, 공통 특성, 세부사항 은닉
