#SW공학 #객체지향 #필수

## 정의
상속성(Inheritance)
상위 클래스의 속성과 기능을 하위 클래스가 물려받아 재사용하는 기법, 상속성
- 부모 클래스(Super Class)의 특징을 자식 클래스(Sub Class)가 공유하도록 설계하는 방식
- 코드의 중복을 제거하고 계층적인 구조를 통해 시스템을 체계적으로 확장함
## 키워드
* 슈퍼클래스, 서브클래스(콘크리트 클래스), 오버라이딩, 다중상속
## 암기법
* Extends
## 구성요소
- 상위 클래스: 상속을 제공하는 부모 클래스 (Base Class, Super Class)
- 하위 클래스: 상속을 받는 자식 클래스 (Concrete Class, Sub Class)
- 메서드 상속: 상위 클래스의 메서드를 하위 클래스에서 사용 가능
- 메서드 오버라이딩: 하위 클래스에서 상위 클래스 메서드를 재정의
- 다중 상속: 여러 상위 클래스로부터 상속받는 것 (일부 언어에서 지원)
- 계층 구조: 클래스 간 상속 관계로 형성되는 계층 구조
## 구성도 요소
* 관계: 상속이 너무 깊어지면 결합도가 높아져 유지보수가 어려워질 수 있음 (상속보다 합성을 권장하기도 함)
```
     [Parent Class]
           ^
           | (extends)
     [Child Class]
```

```
class QnA extends BBS {
	
}
```
