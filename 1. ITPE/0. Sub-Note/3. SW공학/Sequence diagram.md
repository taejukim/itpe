#SW공학 #UML #행위다이어그램

## 정의
시간 순서 상호작용 표현, 시퀀스 다이어그램
- 시스템의 객체들이 메시지를 주고받으며 상호작용하는 과정을 시간 순서대로 표현하는 UML 다이어그램
- 동적 행위를 시각화하여 시스템의 실행 흐름과 객체 간 협력 관계를 명확히 이해
## 키워드
- Lifeline, Activation, Message, Fragment
## 암기법
- **구성**: 객생활메 (객체-생명선-활성화-메시지)
## 구성요소
| 요소 | 표기 | 설명 |
|-----|------|------|
| 객체 | :Class | 상호작용 참여자 |
| 생명선 | │ | 객체 존재 기간 |
| 활성화 | █ | 실행 중인 기간 |
| 메시지 | → | 동기 호출 |
| 응답 | --> | 반환 |

## 특징
- 동적 행위: 객체 간 상호작용 표현
- 시간 축: 위→아래 시간 흐름
- 메시지 중심: 호출/응답 순서 명확
## 메시지 유형
| 유형 | 표기 | 설명 |
|-----|------|------|
| 동기 | ─▶ | 응답 대기 |
| 비동기 | ─> | 응답 미대기 |
| 자기호출 | ↺ | 자신 호출 |
