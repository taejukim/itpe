## 정의
시간 순서 상호작용 표현, 시퀀스 다이어그램
- 시스템의 객체들이 메시지를 주고받으며 상호작용하는 과정을 시간 순서대로 표현하는 UML 다이어그램
- 동적 행위를 시각화하여 시스템의 실행 흐름과 객체 간 협력 관계를 명확히 이해

## 특징
- 시간 순서: 시간 축 기반 표현
- 상호작용: 메시지 교환 중심
- 실행 흐름: 메서드 호출 순서 명시
- 생명선: 객체 생존 기간 표시

## 목적
- 시간 순서에 따른 객체 간 상호작용 표현
- 시스템 실행 흐름 이해 및 협업 패턴 분석

## 구성요소
- Actor/Object(액터/객체): 상호작용 참여자 (상단 박스)
- Lifeline(생명선): 객체의 생존 기간 (수직 점선)
- Activation Bar(활성화 박스): 객체가 활동 중인 기간 (생명선 위의 얇은 사각형)
- Message(메시지): 객체 간 통신 (수평 화살표)
- Synchronous Message(동기 메시지): 응답 대기 (실선 화살표 ▶)
- Asynchronous Message(비동기 메시지): 응답 비대기 (실선 화살표 >)
- Return Message(반환 메시지): 결과 반환 (점선 화살표)
- Self Message(자기 호출): 자신에게 메시지 (자기 자신으로 화살표)
- Frame(프레임): 상호작용 경계 (alt, opt, loop, par 등)

- Actor, Object, Lifeline, Activation Bar
- Message, Synchronous, Asynchronous, Return
- Frame, alt, loop, opt, par
- 시간 순서, 상호작용, 메시지 교환

## 암기법

- 메시지 3종: 동비반 (동기-비동기-반환)
