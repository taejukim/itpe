## 정의
클라우드 서비스 특화 감리, 클라우드 감리
- 클라우드 컴퓨팅 환경의 구축·운영에 대한 적정성을 검증하는 특화 감리
- 서비스 모델(IaaS, PaaS, SaaS)과 배포 모델(Public, Private, Hybrid)의 특성을 고려한 점검

## 구성요소
### 클라우드 특화 점검 영역
- 서비스 모델 적정성: IaaS, PaaS, SaaS 선택 타당성
- 배포 모델 적정성: Public, Private, Hybrid 선택
- 클라우드 아키텍처: 확장성, 가용성, 탄력성 설계
- 마이그레이션: 기존 시스템 → 클라우드 전환 계획
- 보안 및 컴플라이언스: 데이터 보호, 규제 준수
- SLA(Service Level Agreement): 서비스 수준 협약
- 비용 최적화: 과금 모델, 자원 사용 효율성
- 벤더 종속성(Lock-in): 이식성, 호환성 확보

### 점검 항목
- 클라우드 도입 타당성 분석
- 클라우드 서비스 제공자(CSP) 선정 적정성
- 클라우드 네이티브 설계 (마이크로서비스, 컨테이너)
- 데이터 주권 및 국외 이전
- 멀티/하이브리드 클라우드 전략
- 모니터링 및 로깅
- 재해복구(DR) 및 백업
- 성능 및 확장성 테스트

## 구성도
```
[클라우드 감리 프레임워크]

1. 사업 계획
   ├─ 클라우드 도입 타당성
   ├─ 서비스 모델 선택 (IaaS/PaaS/SaaS)
   ├─ 배포 모델 선택 (Public/Private/Hybrid)
   └─ CSP 선정 기준

2. 아키텍처 설계
   ├─ 클라우드 네이티브 설계
   ├─ 확장성·가용성 설계
   ├─ 마이크로서비스 아키텍처
   └─ 컨테이너·오케스트레이션

3. 마이그레이션
   ├─ 마이그레이션 전략 (7R)
   ├─ 데이터 이전 계획
   ├─ 애플리케이션 재설계
   └─ 테스트 및 검증

4. 보안·컴플라이언스
   ├─ 데이터 암호화
   ├─ 접근 제어 (IAM)
   ├─ 규제 준수 (GDPR, 개인정보보호법)
   └─ 보안 감사 로그

5. 운영 관리
   ├─ SLA 관리
   ├─ 비용 최적화
   ├─ 모니터링·로깅
   └─ 인시던트 관리

[서비스 모델별 점검 포인트]

IaaS:
- 인프라 자원 할당 적정성
- 네트워크 구성
- 스토리지 관리

PaaS:
- 플랫폼 선택 적정성
- 개발 도구 및 CI/CD
- 애플리케이션 배포

SaaS:
- 서비스 선정 적정성
- 데이터 통합
- 커스터마이징 범위

[마이그레이션 7R 전략]

1. Rehost (Lift & Shift)
2. Replatform (Lift & Reshape)
3. Repurchase (SaaS 전환)
4. Refactor (재설계)
5. Retire (폐기)
6. Retain (유지)
7. Relocate (재배치)
```

## 키워드
- IaaS, PaaS, SaaS, Public, Private, Hybrid
- CSP, SLA, 마이그레이션, 클라우드 네이티브
- 벤더 종속성, 데이터 주권, 비용 최적화
- AWS, Azure, GCP, 컨테이너, 쿠버네티스

## 암기법
- 서비스 모델: 이파사 (IaaS-PaaS-SaaS)
- 배포 모델: 퍼프하 (Public-Private-Hybrid)
- 점검 영역: 아마보에운 (아키텍처-마이그레이션-보안-SLA-운영)
