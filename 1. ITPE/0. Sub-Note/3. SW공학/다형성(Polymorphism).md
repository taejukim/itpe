## 정의
다형성(Polymorphism)
같은 인터페이스나 메서드 호출로 다양한 구현을 제공하는 객체지향 원칙, 다형성
- 하나의 인터페이스를 통해 다양한 객체를 동일하게 다룰 수 있는 능력
- 런타임에 객체의 실제 타입에 따라 적절한 메서드가 호출되는 동적 바인딩을 지원

## 목적
- 동일한 메시지 호출로도 객체의 실제 타입에 따라 서로 다른 동작을 수행하게 함으로써 코드의 유연성과 추상화 수준 향상
- 오버라이딩과 동적 바인딩을 통해 상위 클래스의 인터페이스를 유지하면서도 하위 클래스에서 다양한 기능을 구현 및 확장
- 구체적인 클래스 타입이 아닌 인터페이스나 추상 클래스에 의존하게 하여 결합도를 낮추고 다형적 프로그래밍 실현

## 구성요소
- 오버로딩(Overloading): 같은 이름의 메서드를 매개변수만 다르게 정의
- 오버라이딩(Overriding): 상위 클래스의 메서드를 하위 클래스에서 재정의
- 동적 바인딩: 런타임에 객체의 실제 타입에 따라 메서드가 결정됨
- 인터페이스 다형성: 인터페이스를 구현한 다양한 클래스 객체를 동일하게 처리
- 상속 다형성: 상속 관계에서 상위 클래스 참조로 하위 클래스 객체를 다룸

## 구성도
인터페이스/상위 클래스 → 다양한 구현 클래스 → 동일한 인터페이스로 호출 → 실제 구현 실행

## 키워드
