## 정의
다형성(Polymorphism)
하나의 인터페이스가 서로 다른 방식으로 동작하게 하는 기법, 다형성
- 동일한 이름의 메시지에 대해 객체마다 각자의 방식으로 반응하도록 설계하는 방식
- 상속 관계에 있는 객체들 사이에서 코드의 유연성과 확장성을 극대화함

## 구성요소
- 오버라이딩(Overriding): 상위 클래스의 메서드를 하위 클래스에서 재정의 (실행 시점 결정)
- 오버로딩(Overloading): 동일한 이름의 메서드를 매개변수 타입이나 개수를 달리하여 정의 (컴파일 시점 결정) - > 
- 동적 바인딩(Dynamic Binding): 실행 시점에 객체의 타입에 따라 실제 호출될 메서드가 결정됨
- 업캐스팅(Upcasting): 하위 클래스 객체를 상위 클래스 타입으로 참조하여 범용성 확보
- 추상 메서드(Abstract Method): 하위 클래스에서 다형성을 구현하도록 강제하는 수단

## 구성도 요소
* 아키텍처: 상위 클래스 타입의 변수가 여러 하위 객체를 참조하여 동일한 메시지 전송
* 관계: 인터페이스를 통한 표준화와 개별 구현의 다양성 공존
```
[Shape (Draw)]
      |
+-----+-----+
|           |
[Circle]    [Square]
(Draw-O)    (Draw-口)
```

## 키워드
* 오버라이딩, 오버로딩, 동적 바인딩, 유연성, 확장성

## 암기법
* 라이딩재정의 (오버라이딩은 재정의)
* 로딩중복정의 (오버로딩은 중복정의)
