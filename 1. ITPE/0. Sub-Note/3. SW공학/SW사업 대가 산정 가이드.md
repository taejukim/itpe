#SW공학 #비용산정 #대가산정

## 정의
소프트웨어 사업 비용 산정 지침, SW 대가 산정 가이드
- 소프트웨어 개발·유지관리·구매 등의 사업 비용을 적정하게 산정하기 위한 정부 고시 가이드
- 과학기술정보통신부 고시로, 기능점수(FP) 기반 산정 방법론 제시
## 키워드
- 기능점수(FP), IFPUG, ILF, EIF, EI, EO, EQ
- 직접비, 간접비, 이윤, FP당 단가
- 품질 요구수준, 재사용률, 업무 복잡도
- 과기정통부 고시, 원가계산
## 암기법
- 기능점수 5유형: 아이이이이 (ILF-EIF-EI-EO-EQ)
- 비용 구성: 직간이 (직접비-간접비-이윤)
- 보정 3요소: 품기재 (품질-기술-재사용)
## 구성요소
### 대가 산정 방법
- 기능점수(FP) 방식: IFPUG 기반 기능점수 측정
- 원가계산 방식: 직접비 + 간접비 + 이윤
- LOC(Lines of Code) 방식: 코드 라인 수 기반 (비권장)
- 전문가 판단: 유사 프로젝트 비교
### 기능점수(FP) 산정
- ILF(Internal Logical File): 내부 논리 파일
- EIF(External Interface File): 외부 인터페이스 파일
- EI(External Input): 외부 입력
- EO(External Output): 외부 출력
- EQ(External inQuiry): 외부 조회
### 비용 산정 공식
```
개발 원가 = 직접비 + 간접비 + 이윤

직접비 = 기능점수(FP) × FP당 단가

FP당 단가 = (기술자 평균 단가 × 생산성) / 재사용률

간접비 = 직접비 × 간접비율 (통상 110~120%)

이윤 = (직접비 + 간접비) × 이윤율 (통상 10~15%)
```
### 보정 계수
- 품질 요구수준: 높음(1.2), 보통(1.0), 낮음(0.8)
- 기술 수준: 신기술(1.2), 일반(1.0)
- 재사용률: 0% ~ 30%
- 업무 복잡도: 상(1.2), 중(1.0), 하(0.8)
## 구성도
```
[FP 기반 대가 산정 프로세스]

1. 기능점수 측정
   ├─ ILF 카운트
   ├─ EIF 카운트
   ├─ EI 카운트
   ├─ EO 카운트
   └─ EQ 카운트
   
2. 미조정 FP 산출
   = Σ(각 기능 × 복잡도 가중치)

3. 조정 FP 산출
   = 미조정 FP × VAF
   (VAF: Value Adjustment Factor)

4. 직접비 산정
   = 조정 FP × FP당 단가

5. 간접비 산정
   = 직접비 × 간접비율

6. 이윤 산정
   = (직접비 + 간접비) × 이윤율

7. 총 개발 비용
   = 직접비 + 간접비 + 이윤

[기능점수 복잡도 가중치]

유형 │ 단순 │ 보통 │ 복잡 │
─────┼──────┼──────┼──────┤
ILF  │  7   │  10  │  15  │
EIF  │  5   │  7   │  10  │
EI   │  3   │  4   │  6   │
EO   │  4   │  5   │  7   │
EQ   │  3   │  4   │  6   │

[산정 예시]

기능점수: 500 FP
FP당 단가: 800,000원

직접비 = 500 × 800,000 = 400,000,000원
간접비 = 400,000,000 × 110% = 440,000,000원
이윤 = (400,000,000 + 440,000,000) × 15%
     = 126,000,000원

총 비용 = 400,000,000 + 440,000,000 + 126,000,000
        = 966,000,000원

[유지보수 대가]

연간 유지보수 비용 = 개발 비용 × 7~10%
```
