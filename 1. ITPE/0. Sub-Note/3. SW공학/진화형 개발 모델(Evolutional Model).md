## 정의
진화형 개발 모델(Evolutional Model)
시스템의 프로토타입을 개발하면서 지속적으로 발전시켜 나가는 방법, 진화형 개발 모델
- 초기 프로토타입에서 시작하여 사용자 피드백을 받아 지속적으로 기능을 추가하고 개선하는 방법론
- 진화에 대한 전체 개요(Outline)가 필요하며, 핵심 기능 개발 후 구성요소를 개선

## 목적
- 요구사항이 불명확하거나 가변적인 환경에서 초기 핵심 기능 위주의 프로토타입을 지속적으로 발전시켜 최종 제품 완성
- 사용자 피드백을 주기적으로 수집하고 반영하는 과정을 통해 최종 시스템의 요구사항 충족도와 사용자 만족도 극대화
- 개발 과정 자체가 요구사항을 탐색하고 명확화하는 과정이 되어 소프트웨어의 비즈니스 적합성(Fitness) 제고

## 구성요소
- 전체 개요: 시스템의 전체적인 구조와 목표 정의
- 프로토타입 개발: 핵심 기능을 포함한 초기 프로토타입 개발
- 사용자 피드백: 사용자로부터 피드백 수집
- 기능 추가: 피드백을 반영하여 새로운 기능 추가
- 구성요소 개선: 기존 구성요소의 성능 및 품질 개선
- 지속적 진화: 시스템을 지속적으로 발전시켜 최종 시스템 완성

## 구성도
전체 개요 → 프로토타입 개발 → 사용자 피드백 → 기능 추가 → 구성요소 개선 → 지속적 진화

## 키워드
- 프로토타입, 지속적 발전, 진화, 피드백, 점진적 개선
