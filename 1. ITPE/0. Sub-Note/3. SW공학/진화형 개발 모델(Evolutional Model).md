## 정의
요구사항을 발전시키며 시스템을 점진적으로 확장하는 방식, 진화형 모델
- 초기에는 핵심 요구사항만 구현하고, 사용자의 피드백을 받아 시스템을 진화(Evolve)시키는 모델
- 요구사항이 매우 불명확하거나 실험적인 프로젝트에 적합함

## 구성요소
- ==핵심 버전(Core Version)==: 가장 기본이 되는 기능을 구현한 첫 번째 산출물
- 사용자 평가 및 피드백: 실제 환경에서 사용 후 개선 사항 도출
- 지속적 진화: 피드백을 바탕으로 아키텍처와 기능을 수정하고 확장
- 프로토타이핑 연계: 프로토타입을 폐기하지 않고 계속 발전시켜 본 시스템으로 전환
- 유연한 설계: 지속적인 확장을 고려한 느슨한 결합의 설계 지향
