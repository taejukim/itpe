## 정의
시스템 분할 설계 기법, 모듈화
- 소프트웨어 시스템을 기능적으로 독립된 모듈로 분할하여 개발, 테스트, 유지보수를 용이하게 하는 설계 기법
- 높은 응집도(High Cohesion)와 낮은 결합도(Low Coupling)를 추구하여 복잡도 관리와 재사용성 향상

## 특징
- 독립성: 모듈 간 의존성 최소화
- 재사용성: 독립 모듈의 다양한 활용
- 유지보수성: 변경 영향 범위 최소화
- 이해용이: 복잡도 감소로 파악 용이

## 목적
- 강한 응집도와 약한 결합도로 시스템 복잡도 관리
- 모듈의 독립성 확보로 재사용성 및 유지보수성 향상

## 구성요소
- Module(모듈): 독립적으로 컴파일/실행 가능한 프로그램 단위
- Cohesion(응집도): 모듈 내부 요소들의 기능적 연관성 (강할수록 좋음)
- Coupling(결합도): 모듈 간 상호 의존성 (약할수록 좋음)
- Interface(인터페이스): 모듈 간 상호작용 명세
- Information Hiding(정보은닉): 모듈 내부 구현 세부사항 숨김
- Abstraction(추상화): 핵심 기능만 외부 노출

## 구성도
```
[모듈화 원칙]

┌──────────────────────────────────────┐
│         High Cohesion (강한 응집)     │
│  ┌─────────────────────────────┐    │
│  │     Module A (독립적)        │    │
│  │  - 기능1  - 기능2  - 기능3   │    │
│  └──────────┬──────────────────┘    │
└─────────────┼───────────────────────┘
              │ Interface (약한 결합)
              ▼
┌─────────────┴───────────────────────┐
│         Low Coupling (약한 결합)     │
│  ┌─────────────────────────────┐    │
│  │     Module B (독립적)        │    │
│  │  - 기능4  - 기능5  - 기능6   │    │
│  └─────────────────────────────┘    │
└──────────────────────────────────────┘

[응집도 수준 (낮음→높음)]
우연적 < 논리적 < 시간적 < 절차적 < 통신적 < 순차적 < 기능적

[결합도 수준 (높음→낮음)]
내용 > 공통 > 외부 > 제어 > 스탬프 > 자료 > 메시지
```

## 키워드
- Module, Cohesion, Coupling, Interface
- Information Hiding, Abstraction, Independence
- 기능적 응집도, 자료 결합도
- 재사용성, 유지보수성, 확장성

## 암기법
- 모듈화 2대 원칙: 강응약결 (강한 응집도, 약한 결합도)
- 응집도: 우논시절통순기 (기능적이 최고)
- 결합도: 내공외제스자메 (메시지가 최선)
