#SW공학 #설계원리 #필수

## 정의
시스템 분할 설계 기법, 모듈화
- 시스템을 분해하고 추상화하여 성능 향상
- 응집도는 높이고, 결합도는 낮춤
## 키워드
- 응집도, 결합도
- 정보 은폐, 자료 추상화
- Fan-in, Fan-out, 캡슐화
## 암기법
- **원리**: 비정자독
- 응집도: 우논시절통순기 (기능적이 최고)
- 결합도: 내공외제스자메 (메시지가 최선)
## 목적
- 강한 응집도와 약한 결합도로 시스템 복잡도 관리
- 모듈의 독립성 확보로 재사용성 및 유지보수성 향상
## 구성도
```
[응집도 수준 (낮음→높음)]
우연적 < 논리적 < 시간적 < 절차적 < 통신적 < 순차적 < 기능적

[결합도 수준 (높음→낮음)]
내용 > 공통 > 외부 > 제어 > 스탬프 > 자료 > 메시지
```

## 특징
- 독립성: 낮은 결합도, 높은 응집도
- 정보 은폐: 변경 가능 모듈 은폐
- 재사용성: 모듈 재사용 가능
## 모듈화 원리
- 비용과 모듈 관계: 적정 모듈 개수 유지
- 정보 은폐: 변경 가능성 있는 모듈 은폐
- 자료 추상화: 자료구조 표현 은폐
- 모듈 독립성: 낮은 결합도, 높은 응집도
## 응집도 (높을수록 좋음)
1. 우연적 < 2. 논리적 < 3. 시간적 < 4. 절차적 < 5. 통신적 < 6. 순차적 < 7. **기능적**(최고)
## 결합도 (낮을수록 좋음)
1. 내용(최악) > 2. 공통 > 3. 외부 > 4. 제어 > 5. 스탬프 > 6. **자료**(최저)
## 장점
- 효율적 관리 및 성능 향상
- 이해 용이성 증대, 복잡성 감소
- 시험, 통합, 수정 용이
- 오류 파급 효과 감소
- 재사용성 증대
