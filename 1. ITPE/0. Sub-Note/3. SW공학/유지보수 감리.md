## 정의
유지보수 활동 적정성 감리, 유지보수 감리
- 정보시스템의 유지보수 계약, 수행, 관리가 적절하게 이루어지는지 점검하는 감리
- 유지보수 범위, 비용, 품질, 대응 시간 등의 적정성을 종합 평가

## 구성요소
### 유지보수 점검 영역
- 계약 관리: 유지보수 범위, 비용, SLA 적정성
- 요청 관리: 요청 접수, 분류, 우선순위 설정
- 변경 관리: 변경 영향 분석, 승인, 구현, 테스트
- 장애 대응: 장애 접수, 대응, 복구, 사후 분석
- 품질 관리: 코드 품질, 테스트, 결함 밀도
- 형상 관리: 버전 관리, 릴리스 관리
- 문서 관리: 매뉴얼 갱신, 이력 관리
- 인력 관리: 유지보수 인력 투입 적정성

### 유지보수 유형
- 완전 유지보수(Perfective): 기능 개선 및 추가
- 적응 유지보수(Adaptive): 환경 변화 대응
- 수정 유지보수(Corrective): 오류 수정
- 예방 유지보수(Preventive): 잠재 결함 제거

### 점검 항목
- 유지보수 계약 범위 명확성
- 대응 시간(Response Time) 준수
- 변경 요청 처리 절차
- 소스코드 인수인계 상태
- 테스트 수행 적정성
- 형상관리 도구 활용
- 문서화 수준
- 사용자 만족도

## 구성도
```
[유지보수 감리 프레임워크]

1. 계약 관리
   ├─ 유지보수 범위 정의
   │  ├─ 포함 범위
   │  └─ 제외 범위
   ├─ SLA 정의
   │  ├─ 대응 시간
   │  └─ 해결 시간
   └─ 비용 산정 적정성

2. 요청 관리
   ├─ 요청 접수 체계
   ├─ 분류 기준 (장애/개선/문의)
   ├─ 우선순위 설정
   └─ 진행 상황 추적

3. 변경 관리
   ├─ 변경 요청서 (RFC)
   ├─ 영향도 분석
   ├─ 변경 승인 위원회 (CAB)
   ├─ 변경 구현 및 테스트
   └─ 배포 및 검증

4. 장애 대응
   ├─ 장애 등급 분류
   │  ├─ Critical: 1시간
   │  ├─ High: 4시간
   │  └─ Normal: 24시간
   ├─ 대응 프로세스
   └─ 사후 분석 (RCA)

5. 품질 관리
   ├─ 코드 리뷰
   ├─ 단위 테스트
   ├─ 회귀 테스트
   └─ 결함 추적

[유지보수 프로세스]

요청 접수 → 분류 → 분석 → 승인 → 구현 → 테스트 → 배포 → 검증
    │                                                      │
    └────────────── 형상관리 ───────────────────────────────┘

[대응 시간 기준 예시]

┌────────┬──────────┬──────────┐
│ 등급   │ 대응시간 │ 해결시간 │
├────────┼──────────┼──────────┤
│Critical│  1시간   │  4시간   │
│High    │  4시간   │  8시간   │
│Medium  │  8시간   │ 24시간   │
│Low     │ 24시간   │ 48시간   │
└────────┴──────────┴──────────┘

Adaptive (적응): 25%

Preventive (예방): 5%
```

## 키워드
- 유지보수 계약, SLA, 대응시간, 변경관리
- RFC, CAB, RCA, 형상관리
- Corrective, Adaptive, Perfective, Preventive
- 소스코드 인수인계, 회귀 테스트

## 암기법
- 유지보수 4유형: 수적완예 (수정-적응-완전-예방)
- 점검 영역: 계요변장품형문인 (계약-요청-변경-장애-품질-형상-문서-인력)
- 프로세스: 접분분승구테배검 (접수-분류-분석-승인-구현-테스트-배포-검증)
