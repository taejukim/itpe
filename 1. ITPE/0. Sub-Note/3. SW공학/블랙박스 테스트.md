## 정의
명세 기반 테스트, 블랙박스 테스트
- 소프트웨어의 내부 구조를 고려하지 않고 기능 명세와 요구사항을 기반으로 수행하는 테스트
- 입력값과 출력값의 관계만 확인하여 외부 사용자 관점에서 기능 동작 검증

## 특징
- 명세 기반: 요구사항 문서 중심
- 사용자 관점: 외부 인터페이스 검증
- 구조 독립: 내부 코드 불필요
- 기능 중심: 기능 동작 확인

## 목적
- 요구사항 대비 기능 동작의 정확성 검증
- 사용자 관점의 소프트웨어 품질 확보

## 구성요소
- Equivalence Partitioning(동등 분할): 입력 도메인을 동등 클래스로 분할
- Boundary Value Analysis(경계값 분석): 경계값과 인근 값 테스트
- Decision Table(결정 테이블): 조건과 행위의 조합 테스트
- State Transition(상태 전이): 상태 변화 시나리오 테스트
- Use Case Testing(유스케이스 테스트): 사용자 시나리오 기반 테스트
- Pairwise Testing(쌍별 테스트): 파라미터 조합 최적화
- Cause-Effect Graph(원인-결과 그래프): 입력 조건과 결과 관계 분석

## 구성도
```
[블랙박스 테스트 개념]

┌─────────────────────────────┐
│     Black Box               │
│  ┌─────────────────────┐    │
│  │   내부 구조 무시     │    │
│  │   (보이지 않음)      │    │
│  └─────────────────────┘    │
└─────────────────────────────┘
   ▲                      │
   │ Input               │ Output
   │                     ▼
검증: 기대 출력 = 실제 출력?

[동등 분할 예시]
나이 입력 (0-150)
├─ 유효 클래스: 1-150
├─ 무효 클래스1: < 0
└─ 무효 클래스2: > 150

[경계값 분석 예시]
범위: 10-100
테스트 값: 9, 10, 11, 99, 100, 101
(경계값: 10, 100)
(경계값±1: 9, 11, 99, 101)

[결정 테이블 예시]
조건1(나이≥18) │ T │ T │ F │ F │
조건2(면허有)   │ T │ F │ T │ F │
───────────────┼───┼───┼───┼───┤
행위(운전가능)  │ O │ X │ X │ X │

[상태 전이 예시]
대기 ──로그인──> 인증 ──승인──> 접속
 │               │
 └───타임아웃─────┘
```

## 키워드
- Equivalence Partitioning, Boundary Value Analysis
- Decision Table, State Transition, Use Case
- 명세 기반, 기능 테스트, 입출력 검증
- 요구사항 추적, 사용자 관점

## 암기법

  (동등분할-경계값-결정테이블-상태전이-유스케이스-쌍별-원인결과)
- 핵심 3가지: 동경결 (동등분할-경계값-결정테이블)
