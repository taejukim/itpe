## 정의
명세 기반 테스트, 블랙박스 테스트
- 주어진 명세를 바탕으로 테스트 케이스를 도출하는 기법
- 프로그램의 요구사항 명세서를 기반으로 실행하는 테스트
## 키워드
- Black Box, 명세 기반
- 동등 분할, 경계값 분석
- Decision Table, State Transition
- Use Case, 기능 테스트
## 암기법
- **주요 5개**: 블동경의상유 (블랙박스 > 동등분할-경계값-의사결정-상태전이-유스케이스)
- **블동경의상**: 불교 동경에 의상대사 가셨다
- **입출력**: I/O 기반 테스트
## 목적
- 요구사항 대비 기능 동작의 정확성 검증
- 사용자 관점의 소프트웨어 품질 확보

## 특징
- 기능 위주: 데이터 기반, I/O 기반
- 명세 기반: 요구사항 분석서, 설계서 활용
- 내부 구조 무관: 코드 구조 몰라도 테스트 가능
## 주요 기법
### 1. 동등 클래스 분할 (Equivalence Partitioning)
- 입력 데이터를 동등한 클래스로 분할
- 각 클래스에서 대표값 선택
- 원칙: 범위보다 작은 값, 범위 내 값, 범위 넘는 값
### 2. 경계값 분석 (Boundary Value Analysis)
- 경계값에서 오류 발생 가능성 높음
- 최솟값, 최댓값, 경계값 테스트
- 동등 분할의 보완 기법
### 3. 의사결정 테이블 (Decision Table Testing)
- 조건과 행위의 조합 테스트
- 복잡한 비즈니스 규칙 검증
### 4. 상태전이 (State Transition Testing)
- 시스템의 상태 변화 테스트
- 유효/무효 전이 검증
### 5. 유스케이스 (Use Case Testing)
- 시나리오 기반 테스트
- 사용자 관점의 종단간 테스트
## 기타 기법
- All-pairs Testing: 조합 최소화
- Cause-Effect Graph: 원인-결과 관계
- Domain Analysis: 도메인 분석
- Syntax Testing: 문법 검증
