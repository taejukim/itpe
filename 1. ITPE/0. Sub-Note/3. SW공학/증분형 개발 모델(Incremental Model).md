## 정의
증분형 개발 모델(Incremental Model)
요구사항의 범위 일부분을 반복적으로 개발하며, Speed를 위해 병렬 개발하면서 대상 범위를 개발하여 최종 제품을 완성하는 방법, 증분형 개발 모델
- 폭포수 모델의 변형으로 하나의 시스템을 여러 서브시스템으로 분리하여 점차 증가시키는 방법론
- 각 증분(Increment)은 독립적으로 개발되고 통합되어 최종 시스템을 완성

## 목적
- 전체 시스템을 독립적인 서브시스템(증분)으로 분리하여 단계적으로 인도함으로써 핵심 기능을 조기에 사용 가능하게 함
- 각 증분을 병렬로 개발하여 전체 개발 기간을 단축하고 초기 증분에서 얻은 피드백을 후속 개발에 반영하여 리스크 감소
- 한꺼번에 거대한 시스템을 구축하는 부담을 줄이고 단계별 투자를 통해 프로젝트의 경제성과 유연성 확보

## 구성요소
- 전체 설계: 시스템 전체의 기본 설계 수립
- 증분 정의: 개발할 증분 단위 정의 및 우선순위 결정
- 증분 개발: 각 증분을 폭포수 모델로 개발
- 통합: 개발된 증분을 기존 시스템에 통합
- 검증: 통합된 시스템 검증 및 다음 증분 계획

## 구성도
전체 설계 → 증분 정의 → 증분 개발 → 통합 → 검증 → 다음 증분 (점진적 증가)

## 키워드
- 증분, 병렬 개발, 점진적 증가, 독립적 개발, 통합
