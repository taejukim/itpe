## 정의
증분형 개발 모델(Incremental Model)
순차적으로 나눠 개발하는 증분형 개발 모델
- 전체 요구사항을 여러 개의 증분(Increment)으로 나누고, 각 증분을 독립적으로 개발하여 출시
- 핵심 기능을 먼저 인도하여 사용자가 조기에 시스템을 사용할 수 있도록 함

## 구성요소
- 증분(Increment): 독립적으로 실행 가능한 소프트웨어의 일부분
- 우선순위 부여: 핵심적이고 가치가 높은 기능을 첫 번째 증분으로 설정
- 병렬 개발 가능: 각 증분의 의존성이 낮을 경우 여러 팀이 동시에 개발 가능
- 단계적 인도: 각 증분이 완성될 때마다 사용자에게 전달하여 즉시 활용 유도
- 통합 관리: 새로운 증분이 추가될 때마다 기존 시스템과의 통합 테스트 수행
