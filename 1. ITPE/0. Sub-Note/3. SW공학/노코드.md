#SW공학 #개발방법론 #로우코드

## 정의
코드 없는 소프트웨어 개발, 노코드
- 프로그래밍 언어 없이 시각적 인터페이스와 ==드래그앤드롭== 방식으로 애플리케이션을 개발하는 방법론
- 비개발자(시민 개발자)도 비즈니스 로직을 구현할 수 있어 개발 속도와 생산성 향상
## 키워드
- Visual Builder, Drag-and-Drop, Pre-built Component, Workflow
## 암기법
- 노코드 3요소: 비드워 (Visual-Drag&Drop-Workflow)
- 핵심 가치: 빠속쉬 (빠른 개발-속도-쉬운 접근)
- Low-Code(조금 코딩) vs No-Code(코딩 없음)
## 목적
- 시민 개발자의 참여로 개발 민주화 실현
- 빠른 프로토타이핑과 시장 출시 시간 단축
## 구성요소
- Visual Builder(시각적 빌더): 드래그앤드롭 기반 UI 구성 도구
- Pre-built Component(사전 구축 컴포넌트): 재사용 가능한 기능 모듈
- Workflow Engine(워크플로우 엔진): 비즈니스 로직 자동화
- Integration(통합): API, 데이터베이스, 외부 서비스 연결
- Template(템플릿): 산업별/기능별 사전 제작 앱
- Citizen Developer(시민 개발자): 비전문 개발자
- Low-Code(로우코드): 최소 코딩 + 시각적 개발 (노코드의 확장형)

## 특징
- 빠른 개발: 직접 코딩 최소화
- 시각적 IDE: 워크플로우, 데이터 모델, UI 모델링
- 낮은 진입 장벽: Citizen Developers도 개발 가능
- One-click 배포
## Low-Code vs No-Code
### Low-Code
- 핸드코딩 최소화
- 시각적 개발: UI/Workflow/API 연동
- IT-Biz간 협업 용이
- 플랫폼: Power Apps, OutSystems, Mendix, Salesforce
### No-Code
- 100% 시각적 개발
- 코딩 지식 전혀 불필요
- 더 제한적인 커스터마이징
- 더 빠른 개발 속도
