## 정의
빅데이터 시스템 특화 감리, 빅데이터 감리
- 빅데이터 플랫폼 구축·운영의 적정성을 검증하는 특화 감리
- 데이터 수집·저장·처리·분석의 전 과정과 데이터 품질, 거버넌스, 보안을 종합 점검

## 구성요소
### 빅데이터 특화 점검 영역
- 데이터 아키텍처: 수집·저장·처리·분석 아키텍처 설계
- 데이터 수집: 다양한 소스로부터 데이터 수집 체계
- 데이터 저장: 분산 파일 시스템, NoSQL, Data Lake
- 데이터 처리: 배치/실시간 처리, ETL/ELT
- 데이터 분석: 분석 모델, 시각화, 머신러닝
- 데이터 품질: 정확성, 완전성, 일관성, 적시성
- 데이터 거버넌스: 정책, 표준, 메타데이터 관리
- 데이터 보안: 암호화, 접근제어, 개인정보보호
- 성능 및 확장성: 대용량 데이터 처리 성능
- 플랫폼 선정: Hadoop, Spark, Kafka 등 적정성

### 점검 항목
- 빅데이터 3V 고려 (Volume, Velocity, Variety)
- 데이터 레이크 아키텍처 설계
- 실시간/배치 처리 아키텍처 (Lambda, Kappa)
- 데이터 파이프라인 구성
- 데이터 품질 관리 체계
- 메타데이터 관리
- 개인정보 비식별화
- 분석 모델 검증
- 시각화 및 대시보드
- 인프라 용량 계획

## 구성도
```
[빅데이터 감리 프레임워크]

1. 아키텍처 설계
   ├─ 수집 계층 (Ingestion Layer)
   │  └─ Kafka, Flume, Logstash
   ├─ 저장 계층 (Storage Layer)
   │  └─ HDFS, HBase, S3
   ├─ 처리 계층 (Processing Layer)
   │  └─ Spark, Flink, MapReduce
   └─ 분석 계층 (Analytics Layer)
      └─ Hive, Presto, ML Libraries

2. 데이터 수집
   ├─ 데이터 소스 식별
   ├─ 수집 방식 (배치/실시간)
   ├─ 데이터 검증
   └─ 오류 처리

3. 데이터 저장
   ├─ 저장 방식 (Raw/Processed/Curated)
   ├─ 데이터 모델링
   ├─ 파티셔닝 전략
   └─ 압축 및 최적화

4. 데이터 처리
   ├─ ETL/ELT 파이프라인
   ├─ 배치 처리 (Spark Batch)
   ├─ 스트림 처리 (Spark Streaming)
   └─ 데이터 변환 및 정제

5. 데이터 품질
   ├─ 정확성 (Accuracy)
   ├─ 완전성 (Completeness)
   ├─ 일관성 (Consistency)
   ├─ 적시성 (Timeliness)
   └─ 유효성 (Validity)

6. 데이터 거버넌스
   ├─ 데이터 정책
   ├─ 데이터 표준
   ├─ 메타데이터 관리
   └─ 데이터 생명주기

7. 데이터 보안
   ├─ 암호화 (전송/저장)
   ├─ 접근 제어 (RBAC)
   ├─ 개인정보 비식별화
   │  ├─ 가명처리
   │  ├─ 총계처리
   │  ├─ 범주화
   │  └─ 마스킹
   └─ 감사 로그

[빅데이터 아키텍처]

Data Source → Ingestion → Storage → Processing → Analytics
   │             │           │          │            │
 다양한        Kafka       HDFS      Spark        BI Tool
 소스                     Data Lake              Dashboard

[Lambda Architecture]

Batch Layer:   정확성 (Historical Data)
Stream Layer:  속도 (Real-time Data)
Serving Layer: 쿼리 (Query Results)
```

## 키워드
- 3V(Volume, Velocity, Variety), Data Lake, ETL/ELT
- Hadoop, Spark, Kafka, HDFS, NoSQL
- 데이터 품질, 데이터 거버넌스, 비식별화
- Lambda Architecture, 배치·실시간 처리

## 암기법
- 빅데이터 3V: 불벨버 (Volume-Velocity-Variety)
- 점검 영역: 아수저처분품거보 (아키텍처-수집-저장-처리-분석-품질-거버넌스-보안)
- 품질 5요소: 정완일적유 (정확-완전-일관-적시-유효)
