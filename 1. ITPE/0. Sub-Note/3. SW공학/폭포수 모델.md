## 정의
폭포수 모델
SW개발의 계획부터 이행까지 단계적, 순차적 접근하는 하향식 접근방법, 폭포수 모델
- 각 단계를 순차적으로 진행하며 이전 단계가 완료되어야 다음 단계로 진행하는 전통적 개발 모델
- 요구사항이 명확하고 변경이 적은 프로젝트에 적합한 선형적 개발 방법론
- 특징: 고전적, 단계적, 순차적, Top-Down, ==후반부 문제 발생 가능성 높음==

## 목적
- 소프트웨어 개발 생명주기(SDLC)를 명확한 단계로 정의하고 순차적으로 수행하여 프로젝트의 관리 가시성 및 통제력 확보
- 각 단계의 종료 시점마다 정형화된 산출물을 생성하고 검토함으로써 대규모 프로젝트의 품질 안정성 보장
- 요구사항이 명확하고 변경이 적은 프로젝트에서 단계별 책임 소재를 분명히 하고 체계적인 공정 관리 실현

## 구성요소
| 단계 | 설명 | 주요 산출물 |
|------|------|-------------|
| 계획 | 프로젝트 계획 수립 | 프로젝트 계획서 |
| 요구분석 | 사용자 요구사항 분석 및 명세 | 요구사항 명세서(SRS) |
| 설계 | 시스템/상세 설계 | 설계 문서(SDD) |
| 개발 | 코딩 및 단위 테스트 | 소스 코드 |
| 시험 | 단위/통합/시스템/인수 테스트 | 테스트 결과서 |
| 설치 | 시스템 설치 및 배포 | 설치 보고서 |
| 운영 | 시스템 운영 및 유지보수 | 유지보수 이력 |

## V모델 (검증과 확인)
```
요구분석 ─────────────────────────── 인수테스트
    ↘                                  ↗
      설계 ───────────────── 시스템테스트
        ↘                      ↗
         상세설계 ────── 통합테스트
            ↘            ↗
              구현 ── 단위테스트
```
