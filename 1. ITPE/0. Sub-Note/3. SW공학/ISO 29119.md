#SW공학 #테스트 #표준 #품질관리 #필수

## 정의
소프트웨어 테스팅 국제 표준, ISO/IEC/IEEE 29119
- 소프트웨어와 시스템 테스팅을 위한 국제 표준으로, 테스트 프로세스와 문서화 방법 정의
- ISO/IEC/IEEE가 공동 제정한 테스트 표준으로, 조직의 테스트 성숙도 향상 지원
## 키워드
- ISO/IEC/IEEE 29119, Test Process, Test Documentation
- Test Policy, Test Strategy, Test Plan
- Test Techniques, Keyword-Driven Testing
- 국제 표준, 테스트 성숙도, 문서화
## 암기법
- 5개 파트: 개프문기키 (개념-프로세스-문서-기법-키워드)
- 프로세스 3계층: 조관행 (조직-관리-실행)
- 관리 3단계: 계통완 (계획-통제-완료)
## 목적
- 테스트 프로세스의 표준화 및 품질 향상
- 조직의 테스트 성숙도 개선 및 일관성 확보
## 구성요소
### Part 1: Concepts and Definitions
- 테스트 개념, 용어, 정의
- 테스트 프로세스 개요
- 이해관계자 역할 정의
### Part 2: Test Processes
- 조직 테스트 프로세스: 정책, 전략
- 테스트 관리 프로세스: 계획, 모니터링, 완료
- 동적 테스트 프로세스: 설계, 구현, 실행
### Part 3: Test Documentation
- 테스트 정책(Test Policy)
- 조직 테스트 전략(Organizational Test Strategy)
- 테스트 계획(Test Plan)
- 테스트 설계 명세(Test Design Specification)
- 테스트 케이스 명세(Test Case Specification)
- 테스트 결과(Test Results)
### Part 4: Test Techniques
- 블랙박스 기법: 동등분할, 경계값, 결정테이블
- 화이트박스 기법: 구문, 결정, 경로 커버리지
- 경험기반 기법: 탐색적, 오류추정
### Part 5: Keyword-Driven Testing
- 키워드 기반 자동화 테스팅
- 테스트 자동화 프레임워크
