## 정의
소프트웨어 개선 3대 기법, 3R
- 레거시 시스템의 개선과 현대화를 위한 세 가지 핵심 기법
- 역공학, 구조 개선, 재공학을 통해 시스템 품질과 유지보수성 향상

## 구성요소
### 1. Reverse Engineering(역공학)
- **정의**: 기존 시스템을 분석하여 설계 정보와 명세 추출
- **목적**: 시스템 이해, 문서화, 재사용
- **산출물**: 설계서, 데이터 모델, 아키텍처 문서
- **도구**: 코드 분석기, UML 생성 도구
- **특징**: 코드 → 설계 (상향식)

### 2. Restructuring(재구조화)
- **정의**: 기능 변경 없이 내부 구조 개선
- **목적**: 가독성, 유지보수성 향상
- **활동**: 코드 정리, 모듈화, 구조 개선
- **산출물**: 개선된 소스코드
- **특징**: 동일 추상화 수준에서 변환

### 3. Reengineering(재공학)
- **정의**: 시스템을 역공학 후 재설계하여 새로운 시스템 구축
- **목적**: 시스템 현대화, 기술 전환
- **프로세스**: 역공학 → 재설계 → 재구현
- **산출물**: 신규 시스템
- **특징**: 가장 포괄적, 비용 높음

```
│ Engineering       │   (코드 → 설계)
└────────┬──────────┘
         │
         ▼
      설계 정보
         │
    ┌────┴────┐
    ▼         ▼

│Restructu│ │Reengineering│
│-ring    │ │             │
└────┬────┘ └──────┬──────┘
     │             │
     ▼             ▼
개선된 코드    신규 시스템

[Reverse Engineering 프로세스]

소스코드
    │
    ▼
코드 분석
    │
    ▼
정보 추출
    │
    ├─ 데이터 구조
    ├─ 제어 흐름
    ├─ 모듈 관계
    └─ 아키텍처
    │
    ▼
문서화

[Restructuring 활동]

Before:
- 스파게티 코드
- 중복 코드
- 긴 함수
- 복잡한 조건문

After:
- 모듈화
- 중복 제거
- 함수 분리
- 명확한 구조

[Reengineering 단계]

1. Reverse Engineering
   └─ 기존 시스템 분석

2. Restructuring
   └─ 설계 개선

3. Forward Engineering
   └─ 새 시스템 구현

[비교표]

구분        │Reverse│Restructure│Reengineering
────────────┼───────┼───────────┼─────────────
방향        │상향식│동일 수준  │하향식 포함
코드 변경   │없음  │있음       │새로 작성
추상화 수준 │상승  │동일       │상승 후 하강
비용        │낮음  │중간       │높음
산출물      │문서  │개선 코드  │신규 시스템
목적        │이해  │개선       │현대화
```

## 키워드
- Reverse Engineering, Restructuring, Reengineering
- 역공학, 재구조화, 재공학
- 레거시 현대화, 리팩토링, Forward Engineering
- 코드 분석, 구조 개선, 시스템 재구축

## 암기법
- 3R: 역재재 (역공학-재구조화-재공학)
- 방향: 상향-동일-전체 (Reverse-Restructure-Reengineering)
- 비용: 낮중높 (역공학 < 재구조화 < 재공학)
