₩#필수
## 정의
시스템 성능 검증, 성능 테스트
- 소프트웨어의 응답시간, 처리량, 자원 사용률 등 비기능 요구사항을 검증하는 테스트
- 부하 상황에서 시스템의 안정성, 확장성, 신뢰성을 측정하여 성능 목표 달성 확인
## 키워드
- Load, Stress, Spike, Endurance, Scalability
- Response Time, Throughput, TPS, Resource
- JMeter, LoadRunner, Gatling, nGrinder
- 병목현상, 튜닝, 최적화
## 암기법
- 성능 테스트 5종: 부스스 내확
  (부하-스트레스-스파이크-내구성-확장성-볼륨)
- 측정 3대 지표: 응처자 (응답시간-처리량-자원사용률)
- 목적: 느린거 찾아서 빠르게 만들기
## 목적
- 시스템 성능 및 확장성 검증
- 병목 지점 식별 및 성능 최적화 방안 도출

## 특징
- 정량화: 수치로 측정 가능
- 부하 기반: 다양한 부하 상황 시뮬레이션
- 비기능: 성능 품질속성 검증
- 지속성: 장시간 테스트 수행
## 성능 테스트 종류
| 종류 | 설명 |
|-----|------|
| Load | 예상 부하에서 성능 측정 |
| Stress | 한계 이상 부하로 안정성 확인 |
| Spike | 급격한 부하 변화 대응 |
| Endurance | 장시간 부하에서 안정성 |
| Scalability | 확장 시 성능 변화 |
## 성능 지표
| 지표 | 설명 |
|-----|------|
| TPS | 초당 트랜잭션 수 |
| Response Time | 응답 시간 |
| Throughput | 처리량 |
| Concurrent Users | 동시 사용자 수 |
