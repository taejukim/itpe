## 정의
시스템 성능 검증, 성능 테스트
- 소프트웨어의 응답시간, 처리량, 자원 사용률 등 비기능 요구사항을 검증하는 테스트
- 부하 상황에서 시스템의 안정성, 확장성, 신뢰성을 측정하여 성능 목표 달성 확인

## 특징
- 정량화: 수치로 측정 가능
- 부하 기반: 다양한 부하 상황 시뮬레이션
- 비기능: 성능 품질속성 검증
- 지속성: 장시간 테스트 수행

## 목적
- 시스템 성능 및 확장성 검증
- 병목 지점 식별 및 성능 최적화 방안 도출

## 구성요소
- Load Testing(부하 테스트): 예상 부하에서 성능 측정
- Stress Testing(스트레스 테스트): 한계 상황까지 부하 증가
- Spike Testing(스파이크 테스트): 급격한 부하 증가 상황 테스트
- Endurance Testing(내구성 테스트): 장시간 지속 부하 테스트
- Scalability Testing(확장성 테스트): 부하 증가 시 확장 능력 측정
- Volume Testing(볼륨 테스트): 대용량 데이터 처리 능력 테스트
- Response Time(응답시간): 요청부터 응답까지 소요 시간
- Throughput(처리량): 단위 시간당 처리 트랜잭션 수
- Resource Utilization(자원 사용률): CPU, 메모리, 네트워크 사용량
