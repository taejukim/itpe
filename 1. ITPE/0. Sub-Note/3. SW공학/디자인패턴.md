디자인패턴(Design Pattern)

#SW공학 #디자인패턴

## 정의
반복문제의 검증된 해결책, 디자인패턴
- GoF(Gang of Four): Erich Gamma 등 4인이 23개 패턴 정의
- 전문가들의 설계 노하우를 정리한 모음집
## 키워드
- GoF, 생구행 (생성-구조-행위)
- Singleton, Factory, Adapter, Façade, Observer, Strategy, Template
- Hollywood Principle
## 암기법
- **생성(5)**: ABProSiFa
- **구조(7)**: ABCDFaFlPro
- **행위(11)**: CCIMM IOSTV
- **중요 7개**: 싱팩어파옵전템
## 목적
- 전문가들의 설계 노하우 공유
- 안정적인 소프트웨어 개발
## 구성요소
- 이름(Name), 분류(생구행), 의도(Intent)
- 구조(Structure), 협력과정, 샘플코드
## 연관 토픽
- [[아키텍처 모델-패턴]] - 시스템 수준
- [[MVC 패턴]] - UI 패턴
- [[객체지향 프로그래밍]] - OOP 기본
## 특징
- 검증된 해결방안 재사용
- 확장성, 재사용성, 유지보수성 향상
- Hollywood Principle: 상위클래스가 하위클래스 호출
## GoF 23가지 패턴
### 생성 패턴 (5개)
- Abstract Factory, Builder, Prototype, ==**Singleton**, **Factory Method**==
- 객체 생성 관련, 무엇을/누가/언제 생성할지 결정
### 구조 패턴 (7개)
- **==Adapter==**, Bridge, Composite, Decorator, **==Façade==**, Flyweight, Proxy
- 클래스 간 구조와 객체 합성
### 행위 패턴 (11개)
- Chain of Responsibility, Command, Interpreter, Iterator, Mediator, Memento
- **==Observer==**, State, **==Strategy==**, **==Template== ==Method==**, Visitor
- 클래스/객체 간 상호작용과 책임 분산
