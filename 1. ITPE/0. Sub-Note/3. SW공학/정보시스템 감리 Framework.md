## 정의
감리 수행 체계 및 방법론, 감리 프레임워크
- 정보시스템 감리를 체계적이고 일관성 있게 수행하기 위한 표준 절차, 기법, 도구의 통합 체계
- 한국지능정보사회진흥원(NIA)의 감리 가이드를 기반으로 감리 프로세스와 점검항목 정의

## 구성요소
### 감리 프로세스
- Planning(계획): 감리 목표, 범위, 일정, 자원 계획
- Execution(실행): 문서검토, 현장점검, 인터뷰 수행
- Analysis(분석): 발견사항 분석, 지적사항 도출
- Reporting(보고): 감리보고서 작성 및 발표
- Follow-up(후속조치): 개선조치 확인

### 감리 점검 체계 (5개 영역)
- 사업관리(PM): 계획, 일정, 비용, 위험, 의사소통
- 요구사항(RQ): 요구 정의, 추적, 변경관리
- 아키텍처/설계(AD): 아키텍처, 상세설계, 인터페이스
- 구현/테스트(IT): 코딩, 단위·통합·시스템 테스트
- 전환/교육(TO): 데이터 전환, 사용자 교육

### 감리 기법
- 문서 검토(Document Review): 산출물 적정성 검토
- 현장 점검(Site Inspection): 현장 확인
- 인터뷰(Interview): 이해관계자 면담
- 체크리스트(Checklist): 점검항목 확인
- 도구 활용(Tool): 소스코드 분석, PMD, SonarQube

## 구성도
```
┌──────────────────────────────────────┐
│     정보시스템 감리 Framework        │
└──────────────────────────────────────┘

[감리 프로세스]

Planning (계획)
  ├─ 감리 계획서
  ├─ 감리팀 구성
  └─ 체크리스트 준비

Execution (실행)
  ├─ 문서 검토
  ├─ 현장 점검
  ├─ 인터뷰
  └─ 산출물 확인

Analysis (분석)
  ├─ 발견사항 정리
  ├─ 지적사항 도출
  └─ 개선방안 수립

Reporting (보고)
  ├─ 감리보고서
  ├─ 결과 브리핑
  └─ 조치계획 수립

Follow-up (후속)
  └─ 조치 이행 확인

[점검 영역 및 항목]

1. 사업관리 (PM)
   - 프로젝트 계획 적정성
   - 일정/비용 관리
   - 위험 관리
   - 의사소통

2. 요구사항 (RQ)
   - 요구사항 정의 명확성
   - 추적성 확보
   - 변경관리 절차

3. 아키텍처/설계 (AD)
   - 아키텍처 설계 적절성
   - 인터페이스 설계
   - 데이터베이스 설계

4. 구현/테스트 (IT)
   - 코딩 표준 준수
   - 테스트 계획/실행
   - 결함 관리

5. 전환/교육 (TO)
   - 데이터 마이그레이션
   - 사용자 교육
   - 시스템 전환 계획
```

## 키워드
- Planning, Execution, Analysis, Reporting, Follow-up
- PM, RQ, AD, IT, TO
- 문서검토, 현장점검, 인터뷰, 체크리스트
- NIA, 감리 가이드, 표준 프로세스

## 암기법
- 프로세스 5단계: 계실분보후 (계획-실행-분석-보고-후속)
- 점검 영역 5개: 사요아구전 (사업-요구-아키-구현-전환)
- 감리 기법: 문현인체도 (문서-현장-인터뷰-체크리스트-도구)
