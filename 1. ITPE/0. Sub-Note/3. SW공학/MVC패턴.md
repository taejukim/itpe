#SW공학 #디자인패턴 #아키텍처 #필수

## 정의
사용자 인터페이스 분리 패턴, MVC
- 애플리케이션을 Model(데이터), View(화면), Controller(제어)로 분리하는 소프트웨어 디자인 패턴
- 비즈니스 로직과 프레젠테이션 로직을 분리하여 유지보수성과 확장성을 향상
## 키워드
- Model, View, Controller, 관심사 분리
## 암기법
- **MVC**: 모뷰컨 (Model-View-Controller)
## 목적
- 비즈니스 로직과 UI 로직의 명확한 분리
- 유지보수성 및 확장성 향상을 통한 개발 효율성 증대
## 구성요소
| 구성요소 | 역할 |
|---------|------|
| Model | 데이터, 비즈니스 로직 |
| View | 화면 표시, UI |
| Controller | 입력 처리, Model/View 연결 |

## 특징
- 관심사 분리: UI와 비즈니스 로직 분리
- 유지보수성: 독립적 개발/테스트 가능
- 재사용성: 컴포넌트 재사용
## 동작 흐름
```
사용자 → Controller → Model → View → 사용자
```
## MVC vs MVVM vs MVI
| 구분 | MVC | MVVM | MVI |
|-----|-----|------|-----|
| 연결 | Controller | ViewModel | Intent |
| 바인딩 | 수동 | 양방향 | 단방향 |
