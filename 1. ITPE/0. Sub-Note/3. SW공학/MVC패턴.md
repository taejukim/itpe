## 정의
사용자 인터페이스 분리 패턴, MVC
- 애플리케이션을 Model(데이터), View(화면), Controller(제어)로 분리하는 소프트웨어 디자인 패턴
- 비즈니스 로직과 프레젠테이션 로직을 분리하여 유지보수성과 확장성을 향상

## 특징
- 관심사 분리: 각 계층의 독립성 확보
- 재사용성: Model과 View 독립적 재사용
- 병렬 개발: 계층별 동시 개발 가능
- 테스트 용이: 계층별 독립 테스트

## 목적
- 비즈니스 로직과 UI 로직의 명확한 분리
- 유지보수성 및 확장성 향상을 통한 개발 효율성 증대

## 구성요소
- Model(모델): 데이터와 비즈니스 로직 담당, DB와 상호작용
- View(뷰): 사용자 인터페이스, 데이터를 시각적으로 표현
- Controller(컨트롤러): 사용자 입력 처리, Model과 View 중재
- 데이터 흐름: User → Controller → Model → Controller → View → User
- 관찰자 패턴: Model 변경 시 View에 자동 통지

## 구성도
```
┌──────────────────────────────────────────┐
│              MVC Pattern                 │
└──────────────────────────────────────────┘

         ┌──────────┐
         │   User   │
         └─────┬────┘
               │ 입력
               ▼
        ┌─────────────┐
        │ Controller  │
        │  (제어로직)  │
        └──┬───────┬──┘
           │       │
    갱신요청 │       │ 데이터조회
           ▼       ▼
    ┌────────┐  ┌────────┐
    │ Model  │  │  View  │
    │(데이터) │  │ (화면)  │
    └────────┘  └────┬───┘
           │         │
           └─통지────►│
                     ▼
                  화면출력

[데이터 흐름]
1. User → Controller: 사용자 입력
2. Controller → Model: 데이터 처리 요청
3. Model → Controller: 처리 결과 반환
4. Controller → View: 화면 갱신 요청
5. View → User: 결과 표시
```

## 키워드
- Model, View, Controller, Separation of Concerns
- 관찰자 패턴, 비즈니스 로직 분리, UI 독립성
- 유지보수성, 재사용성, 확장성

## 암기법
- MVC: 모뷰컨 (모델-뷰-컨트롤러)
- 데이터 흐름: 유→컨→모→컨→뷰→유 (User부터 순환)
- 핵심: 관심사의 분리(Separation of Concerns)
