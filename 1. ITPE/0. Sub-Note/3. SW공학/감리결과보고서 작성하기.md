#SW공학 #감리 #보고서 #필수

## 정의
감리 결과 문서화, 감리결과보고서
- 감리 수행 결과를 체계적으로 정리하여 발주기관에 제출하는 공식 보고 문서
- 지적사항, 개선방안, 종합 의견을 포함하며 사업의 품질 개선과 의사결정 근거로 활용
## 구성요소
### 보고서 구조
1. 종합 의견 : 적합/부적합으로 평가
	1. 전제조건
	2. 총평
	3. 감리영역별 상세점검 결과 요약
2. 과업내용 이행여부 점검결과
3. 감리영역별 점검 결과
	1. 감리 영역 N
4. 별첨
	1. 감리계획서

## 특징
- 공식 문서: 법적 효력
- 종합 보고: 전체 점검 결과
- 개선 중심: 실행 가능한 개선방안
## 감리 보고서 종류
- 설계 단계 감리 보고서
	- 구축하기 전 설계도가 요구사항을 제대로 반영했는지 감리
		- 요구사항 추적성
		- 아키텍처 적절설
		- 데이터베이스 설계
	- 목적 : 예방주사
- 종료 단계 감리 보고서
	- 완료 직전, 목표대로 만들어졌는지 감리
		- 과업 대비 이행도
		- 결함 조치 현황
		- 보안 및 성능
	- 목적 : 시스템 오픈 여부를 결정
