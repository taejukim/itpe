## 정의
구조 기반 테스트, 화이트박스 테스트
- 소프트웨어의 내부 구조와 로직을 기반으로 코드의 실행 경로를 검증하는 테스트
- 소스코드를 직접 확인하여 구문, 분기, 조건, 경로 등의 커버리지를 측정

## 특징
- 구조 기반: 소스코드 직접 분석
- 논리 검증: 내부 로직 확인
- 커버리지: 정량적 측정 가능
- 개발자 관점: 구현 중심 테스트

## 목적
- 코드 내부 로직의 정확성 및 완전성 검증
- 실행 경로 커버리지 확보로 결함 발견률 향상

## 구성요소
- Statement Coverage(구문 커버리지): 모든 문장 최소 1회 실행
- Decision Coverage(결정 커버리지): 모든 분기(참/거짓) 실행
- Condition Coverage(조건 커버리지): 모든 조건식의 참/거짓 실행
- Multiple Condition Coverage(다중 조건): 조건의 모든 조합 실행
- Path Coverage(경로 커버리지): 모든 실행 경로 커버
- Loop Testing(루프 테스트): 반복문의 경계 조건 테스트
- Data Flow Testing(데이터 흐름): 변수 정의와 사용 관계 테스트
- Control Flow Graph(제어 흐름 그래프): 프로그램 흐름 시각화

## 구성도
```
[화이트박스 테스트 개념]

┌─────────────────────────────┐
│     White Box               │
│  ┌─────────────────────┐    │
│  │   내부 구조 확인     │    │
│  │   if(x>0)           │    │
│  │     y=1;            │    │
│  │   else              │    │
│  │     y=0;            │    │
│  └─────────────────────┘    │
└─────────────────────────────┘
검증: 모든 경로 실행 확인

[제어 흐름 그래프]
    ①
    │
    ▼
   ┌─┐
   │②│ if (x>0)
   └┬┘
   ╱ ╲
  Y   N
 ╱     ╲
③       ④
│       │
└───┬───┘
    ▼
    ⑤

[커버리지 비교]
코드 예시:
if (A && B)
    statement1;
else
    statement2;

구문 커버리지: 50% (1개 경로만 실행)
결정 커버리지: 50% (참/거짓 중 1개)
조건 커버리지: 100% (A, B 각각 T/F)
경로 커버리지: 50% (2개 경로 중 1개)

[커버리지 수준]
구문(40%) < 결정(60%) < 조건(70%) < 
다중조건(85%) < 경로(100%)
```

## 키워드
- Statement, Decision, Condition, Path Coverage
- Control Flow Graph, Loop Testing, Data Flow
- 코드 기반, 구조 테스트, 커버리지
- 개발자 관점, 논리 검증

## 암기법
- 커버리지 5단계: 구결조다경 
  (구문-결정-조건-다중조건-경로)
- 강도: 구<결<조<다<경 (경로가 가장 강함)
- 특징: 내부 들여다보고 경로 확인
