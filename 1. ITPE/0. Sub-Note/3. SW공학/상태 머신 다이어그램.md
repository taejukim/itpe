#SW공학 #UML #행위다이어그램

## 정의
객체 상태 전이 표현, 상태 머신 다이어그램
- 객체의 생명주기 동안 이벤트에 따른 상태(State) 변화와 전이(Transition)를 표현하는 UML 다이어그램
- 유한 상태 기계(Finite State Machine)를 기반으로 반응형 시스템의 동적 행위 모델링
## 키워드
- State, Transition, Event, Action
- Initial State, Final State, Guard Condition
- Entry, Exit, Do Activity
- FSM(Finite State Machine), 상태 전이
- 생명주기, 반응형 시스템
## 암기법
- 상태 요소: 상전이가 (State-Transition-Event-Guard-Action)
- 액션 3종: 엔엑두 (Entry-Exit-Do)
- 핵심: 이벤트로 상태 전이
## 목적
- 객체의 생명주기 동안 상태 변화 모델링
- 이벤트 기반 시스템의 동작 명세 및 검증
## 구성요소
- State(상태): 객체가 가질 수 있는 조건/상황 (둥근 모서리 사각형)
- Initial State(초기 상태): 시작 상태 (검은 원)
- Final State(종료 상태): 최종 상태 (테두리 원 안의 검은 원)
- Transition(전이): 상태 간 변화 (화살표)
- Event(이벤트): 전이를 유발하는 사건
- Guard Condition(가드 조건): 전이 발생 조건 [조건]
- Action(액션): 전이 시 수행되는 동작 /action
- Entry Action(진입 액션): 상태 진입 시 수행 (entry/)
- Exit Action(탈출 액션): 상태 이탈 시 수행 (exit/)
- Do Activity(내부 활동): 상태 내에서 지속적으로 수행 (do/)

         │
         ▼
