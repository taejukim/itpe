## 정의
객체지향 프로그래밍
데이터와 로직을 객체로 통합하여 프로그램을 구성하는 프로그래밍 패러다임, 객체지향 프로그래밍
- 현실 세계의 사물을 객체로 모델링하여 소프트웨어를 개발하는 방법론
- 코드의 재사용성, 유지보수성, 확장성을 향상시키기 위한 프로그래밍 접근법

## 구성요소
| 특성 | 설명 | 참조 |
|------|------|------|
| 캡슐화 | 데이터와 메서드를 하나로 묶어 은닉 | [[캡슐화(encapsulation)]] |
| 추상화 | 복잡한 시스템을 단순화하여 핵심만 표현 | [[추상화(abstraction)]] |
| 다형성 | 같은 인터페이스로 다양한 구현 제공 | [[다형성(Polymorphism)]] |
| 상속성 | 기존 클래스의 속성과 메서드를 상속 | [[상속성(Inheritance)]] |
| 정보은닉 | 내부 구현을 숨기고 인터페이스만 노출 | [[정보은닉(Information Hiding)]] |
| 설계 원리 | SOLID 원칙 등 | [[객체지향 설계의 원리]] |

## 구성도
```
[객체지향 핵심 개념]

+------------------+     +------------------+     +------------------+
|     캡슐화       |     |     추상화       |     |     다형성       |
| (데이터+메서드)  |     | (복잡성 감소)    |     | (동일 인터페이스)|
+--------+---------+     +--------+---------+     +--------+---------+
         |                        |                        |
         +------------------------+------------------------+
                                  |
                    +-------------+-------------+
                    |                           |
            +-------v-------+           +-------v-------+
            |    상속성     |           |   정보은닉   |
            | (코드 재사용) |           | (접근 제어)  |
            +---------------+           +---------------+

[객체 구조]
+-----------------------+
|       클래스          |
+-----------------------+
| - 속성 (Attributes)   |
|   - private 변수      |
|   - protected 변수    |
+-----------------------+
| + 메서드 (Methods)    |
|   + getter()          |
|   + setter()          |
|   + operation()       |
+-----------------------+
            |
            | 인스턴스화
            v
+-----------------------+
|       객체            |
| (메모리에 생성된 실체)|
+-----------------------+

[상속과 다형성]
       +---------------+
       |   Animal      |  <-- 부모 클래스 (추상)
       | + makeSound() |
       +-------+-------+
               |
       +-------+-------+
       |               |
+------v------+ +------v------+
|    Dog      | |    Cat      |  <-- 자식 클래스
| makeSound() | | makeSound() |
| (멍멍)      | | (야옹)      |
+-------------+ +-------------+
       다형성: 같은 메서드, 다른 동작
```
