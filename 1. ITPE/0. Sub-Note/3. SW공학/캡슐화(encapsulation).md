## 정의
캡슐화(encapsulation)
데이터와 메서드를 하나의 객체로 묶어 데이터를 보호하고 접근을 제어하는 객체지향 원칙, 캡슐화
- 객체의 내부 데이터를 외부에서 직접 접근하지 못하도록 숨기고 메서드를 통해서만 접근하도록 하는 기법
- 데이터의 무결성을 보장하고 객체 간 결합도를 낮추어 유지보수성을 향상

## 목적
- 연관된 데이터와 메서드를 하나의 논리적 단위인 객체로 묶어 높은 응집도를 유지하고 객체의 독립성 강화
- 내부 데이터에 대한 직접적인 접근을 차단하고 공개된 메서드(인터페이스)를 통해서만 상호작용하게 하여 정보은닉 실현
- 객체 내부의 변경이 외부에 미치는 파급 효과를 최소화함으로써 코드의 수정과 확장이 용이한 구조 구축

## 구성요소
- 데이터 은닉: 객체의 내부 데이터를 private으로 선언하여 외부 접근 차단
- 접근 제어: public, private, protected 등의 접근 제어자를 통한 접근 제어
- 메서드 제공: 데이터에 접근하기 위한 getter/setter 메서드 제공
- 인터페이스: 객체와 상호작용하기 위한 공개 인터페이스 정의
- 데이터 보호: 외부에서 데이터를 직접 수정하지 못하도록 보호

## 구성도
객체 내부 데이터 → 접근 제어 → 메서드 제공 → 외부 인터페이스

## 키워드
- 데이터 은닉, 접근 제어, getter/setter, 인터페이스, 데이터 보호
