## 정의
소프트웨어 결함 검증, 소프트웨어 테스팅
- 소프트웨어가 요구사항을 만족하고 결함이 없는지 확인하는 검증(Verification)과 확인(Validation) 활동
- 품질 보증을 위해 계획-설계-실행-결과분석의 체계적 프로세스로 수행

## 특징
- 체계성: 계획적이고 조직적 수행
- 독립성: 개발과 분리된 활동
- 정량화: 측정 가능한 결과 산출
- 반복성: 지속적 테스트 수행

## 목적
- 소프트웨어 결함 조기 발견 및 품질 보증
- 요구사항 충족 검증 및 사용자 만족도 향상

## 구성요소
- Test Plan(테스트 계획): 범위, 접근법, 자원, 일정 정의
- Test Case(테스트 케이스): 입력값, 실행조건, 예상결과 명세
- Test Scenario(테스트 시나리오): 여러 테스트 케이스의 집합
- Test Data(테스트 데이터): 테스트 실행에 필요한 데이터
- Test Script(테스트 스크립트): 자동화된 테스트 절차
- Defect/Bug(결함): 기대 결과와 실제 결과의 차이
- Test Report(테스트 보고서): 테스트 결과 요약 및 분석

### 테스트 레벨
- Unit Test(단위 테스트): 개별 모듈/함수 테스트
- Integration Test(통합 테스트): 모듈 간 인터페이스 테스트
- System Test(시스템 테스트): 전체 시스템 기능 테스트
- Acceptance Test(인수 테스트): 사용자 요구사항 충족 확인

## 구성도
```
[테스트 프로세스]

┌──────────────┐
│ Test Plan    │ ← 계획 수립
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ Test Design  │ ← 케이스 설계
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ Test Execute │ ← 실행
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ Test Report  │ ← 결과 분석
└──────────────┘

[테스트 레벨 (V-Model)]

요구사항분석 ◄───────► 인수 테스트
    │                     ▲
    ▼                     │
시스템설계   ◄───────► 시스템 테스트
    │                     ▲
    ▼                     │
상세설계    ◄───────► 통합 테스트
    │                     ▲
    ▼                     │
   구현     ────────► 단위 테스트

[테스트 기법 분류]

테스트 기법
├─ Black Box (명세 기반)
│  └─ 동등분할, 경계값, 결정테이블
├─ White Box (구조 기반)
│  └─ 구문, 결정, 조건, 경로 커버리지
└─ Experience (경험 기반)
   └─ 탐색적, 오류추정
```

## 키워드
- Verification, Validation, Test Case, Defect
- Unit, Integration, System, Acceptance
- Black Box, White Box, Coverage
- 결함 발견, 품질 보증, 신뢰성

## 암기법
- 테스트 레벨: 단통시인 (단위-통합-시스템-인수)
- V&V: 검증(Verification)-확인(Validation)
- 테스트 기법: 블화경 (Black-White-Experience)
