## 정의
변경 영향 확인 테스트, 회귀 테스트
- 소프트웨어 수정 후 기존 기능에 부작용(Side Effect)이 발생하지 않았는지 확인하는 재테스트
- 코드 변경, 패치, 업그레이드 시 이전에 정상 동작하던 기능의 무결성 보장
## 키워드
- Regression, Side Effect, Retest, Baseline
- Test Automation, Impact Analysis, Test Suite
- 무결성, 안정성, 지속적 검증
- CI/CD, Selenium, Jenkins
## 암기법
- 회귀 테스트 3단계: 영선실 (영향분석-선택-실행)
- 전략 2가지: 전선 (전체-선택적)
- 핵심: 고친 후 기존 기능 안망가졌나 확인
## 목적
- 코드 변경 후 기존 기능의 무결성 검증
- 부작용 조기 발견으로 품질 저하 방지
### 회귀 테스트 기법
- Retest All(전체 재테스트): 모든 테스트 케이스 재실행
- Selective Retest(선택적 재테스트): 영향받는 부분만 재실행
### 구성요소
- Test Suite(테스트 스위트): 회귀 테스트용 케이스 집합
- Impact Analysis(영향 분석): 변경이 미치는 범위 파악
- Test Automation(테스트 자동화): 반복 실행을 위한 자동화
- Regression Test Selection(테스트 선택): 최소 테스트 세트 선정
- Change Log(변경 이력): 수정 내역 추적
- Baseline(기준선): 이전 버전의 기준 결과
