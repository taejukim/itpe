## 정의
정보은닉(Information Hiding)
객체의 내부 구현 세부사항을 숨기고 외부에는 인터페이스만 노출하는 객체지향 원칙, 정보은닉
- 객체의 내부 데이터 구조와 구현 방법을 외부에 숨겨 결합도를 낮추는 기법
- 인터페이스를 통해서만 객체와 상호작용하도록 하여 유지보수성과 재사용성을 향상

## 목적
- 객체의 내부 구현 세부사항을 외부로부터 격리하여 내부 로직 변경이 외부 호출자에게 영향을 미치지 않도록 설계의 결합도 최소화
- 접근 제어자를 통해 데이터의 무단 수정을 방지하고 객체의 상태를 보호함으로써 시스템의 무결성과 안전성 확보
- 복잡한 내부 동작은 숨기고 단순화된 인터페이스만을 노출하여 객체의 사용 편의성과 재사용성 극대화

## 구성요소
- 구현 은닉: 객체의 내부 구현 세부사항을 숨김
- 인터페이스 노출: 외부와의 상호작용을 위한 공개 인터페이스만 노출
- 접근 제어: private, protected, public 등의 접근 제어자를 통한 정보 보호
- 캡슐화: 데이터와 메서드를 함께 묶어 정보 은닉 실현
- 결합도 감소: 구현 세부사항에 의존하지 않아 객체 간 결합도 감소

## 구성도
객체 내부 구현 → 정보 은닉 → 인터페이스 노출 → 외부 접근

## 키워드
- 구현 은닉, 인터페이스 노출, 접근 제어, 캡슐화, 결합도 감소
