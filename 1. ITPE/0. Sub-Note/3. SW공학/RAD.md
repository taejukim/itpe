## 정의
RAD
강력한 소프트웨어 개발 도구를 이용하여 매우 짧은 주기로 개발을 진행하는 순차적 소프트웨어 개발 프로세스, Rapid Application Development
- CASE 도구를 활용하여 빠른 개발 기간(60-90일) 내에 시스템을 개발하는 방법론
- 사용자의 적극적인 참여를 통해 요구사항을 빠르게 반영하고 프로토타입을 통해 검증

## 목적
- 강력한 CASE 도구와 4세대 언어(4GL)를 활용하여 소프트웨어 개발 주기를 단축하고 시장 출시 속도 극대화
- 사용자의 적극적인 참여(JRP, JAD)와 지속적인 프로토타이핑을 통해 요구사항의 정확성을 높이고 개발 리스크 최소화
- 정형화된 긴 개발 프로세스를 간소화하여 비즈니스 변화에 기민하게 대응할 수 있는 고속 개발 환경 구축

## 구성요소
| 단계           | 영문                         | 설명                            |
| ------------ | -------------------------- | ----------------------------- |
| JRP          | Joint Requirement Planning | 공동 요구사항 계획 - 요구분석             |
| JAD          | Joint Application Design   | 공동 애플리케이션 설계 - 프로토타입 개발/평가/반복 |
| Construction | Construction               | 개발 - 실제 시스템 구축                |
| Cutover      | Cutover                    | 전환 - 테스트, 인수, 운영 전환           |
