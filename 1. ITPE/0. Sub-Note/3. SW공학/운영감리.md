## 정의
정보시스템 운영 단계 감리, 운영감리
- 정보시스템이 안정적으로 운영되고 있는지 적정성을 점검하는 감리
- 서비스 가용성, 성능, 보안, 변경관리, 장애관리 등 운영 전반의 품질 확인

## 구성요소
### 운영 점검 영역
- 서비스 관리: 가용성, 성능, 용량, SLA 준수
- 장애 관리: 장애 대응, 복구, 재발 방지
- 변경 관리: 변경 요청, 승인, 배포, 검증
- 보안 관리: 접근 통제, 보안 모니터링, 취약점 관리
- 백업·복구: 백업 계획, 정기 백업, 복구 테스트
- 자산 관리: H/W, S/W 자산 관리
- 모니터링: 시스템 감시, 로그 관리
- 문서 관리: 운영 매뉴얼, 절차서, 이력 관리

### 점검 항목
- 운영 조직 및 역할 정의
- 운영 프로세스 수립 및 준수
- SLA 목표 달성도
- 장애 발생률 및 복구 시간
- 변경 성공률
- 보안 사고 대응 체계
- 백업 성공률 및 복구 테스트
- 운영 자동화 수준

## 구성도
```
[운영감리 프레임워크]

1. 서비스 관리
   ├─ SLA 관리
   │  ├─ 가용성 목표 (99.9%)
   │  ├─ 응답시간 목표
   │  └─ 처리량 목표
   ├─ 성능 관리
   │  ├─ CPU/Memory 사용률
   │  ├─ 응답시간 모니터링
   │  └─ 병목현상 분석
   └─ 용량 관리
      ├─ 용량 계획
      └─ 자원 증설 기준

2. 장애 관리
   ├─ 장애 분류 체계
   ├─ 장애 대응 프로세스
   │  └─ 발견→분류→대응→복구→분석
   ├─ 장애 복구 시간 (MTTR)
   └─ 재발 방지 대책

3. 변경 관리
   ├─ 변경 요청 프로세스
   ├─ 영향도 분석
   ├─ 변경 승인 체계
   ├─ 배포 계획 및 실행
   └─ 사후 검증

4. 보안 관리
   ├─ 접근 통제 (IAM)
   ├─ 보안 모니터링
   ├─ 취약점 점검
   └─ 보안 사고 대응

5. 백업·복구
   ├─ 백업 정책 (Full/Incremental/Differential)
   ├─ 백업 주기 및 보관
   ├─ 복구 절차
   └─ 복구 테스트 (정기적)

[장애 관리 프로세스]

발견 → 접수 → 분류 → 할당 → 대응 → 복구 → 종료
  │                              │       │
  └─────── 모니터링 ─────────────┘       │
                                        │
                           재발방지 분석 ─┘

[SLA 측정 지표]

가용성 = (전체시간 - 장애시간) / 전체시간 × 100%
MTBF = 총 운영시간 / 장애 발생 횟수
MTTR = 총 복구시간 / 장애 발생 횟수

목표 예시:
- 가용성: 99.9% 이상
- MTBF: 720시간 이상
- MTTR: 4시간 이내
```

## 키워드
- SLA, 가용성, 성능, 장애관리, 변경관리
- MTBF, MTTR, 백업·복구, 모니터링
- ITIL, ITSM, 운영 프로세스
- 안정성, 연속성, 보안

## 암기법
- 운영 5대 영역: 서장변보백 (서비스-장애-변경-보안-백업)
- 장애 프로세스: 발접분할대복종 (발견-접수-분류-할당-대응-복구-종료)
- SLA 3대 지표: 가용성-MTBF-MTTR
