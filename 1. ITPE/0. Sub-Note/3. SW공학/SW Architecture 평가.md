#SW공학 #아키텍처 #평가 #필수

## 정의
아키텍처 품질속성 평가, 소프트웨어 아키텍처 평가
- 소프트웨어 아키텍처가 요구되는 품질속성(성능, 보안, 가용성 등)을 만족하는지 검증하는 프로세스
- 아키텍처 설계 초기에 위험요소를 식별하고 개선방안을 도출하여 프로젝트 리스크 최소화
## 키워드
- ATAM, CBAM, SAAM, 품질 속성, 트레이드오프
## 암기법
- **평가 3기법**: ACS (ATAM-CBAM-SAAM)
## 목적
- 아키텍처 설계의 품질 검증 및 위험 조기 발견
- 품질속성 간 트레이드오프 분석 및 최적 의사결정 지원
## 연관 토픽
- [[SW Architecture]] - [[ISO-IEC-IEEE 42010]]
## 특징
- 사전 예방: 구현 전 설계 단계에서 평가
- 정량화: 시나리오 기반 측정 가능한 평가
- 독립성: 제3자 관점의 객관적 평가
- 트레이드오프: 상충되는 품질속성 균형 분석
## 평가 기법
### ATAM (Architecture Tradeoff Analysis Method)
| 항목 | 내용 |
|-----|------|
| 목적 | 품질 속성 간 트레이드오프 분석 |
| 산출물 | 민감점, 트레이드오프 포인트 |
| 특징 | 시나리오 기반, 품질 속성 중심 |
### CBAM (Cost Benefit Analysis Method)
| 항목 | 내용 |
|-----|------|
| 목적 | 비용-편익 분석 |
| 산출물 | ROI, 우선순위 |
| 특징 | 경제적 관점, ATAM 확장 |
### SAAM (Software Architecture Analysis Method)
| 항목 | 내용 |
|-----|------|
| 목적 | 변경 용이성 평가 |
| 산출물 | 시나리오 기반 평가 |
| 특징 | 초기 평가 기법 |
